formura-egison DSL ギャラリー .fe で書かれた 19 例

DSL で書かれたシミュレーションだけを集めたページです。v1 = 拡張子 .fe の表層 DSL: field E : vector・field B : 2-form と宣言し、物理を添字方程式 (E'_i = E_i + dt * (curl B)_i)や微分形式(B' = B - dt * d E')で書くと、 コンパイラ fec(Haskell、cabal build) が埋め込み形(.egi)へ変換し、Egison の CAS が展開して .fmr を生成、 Formura が MPI+temporal blocking つき C に落とします。カードのコードは .fe(既定で展開表示)→ 中間 .egi → 生成 .fmr の3段。意味保存は .fmr 比較で検証済み (15 .fe 例中6例バイト一致・残りは整形差のみで同値検証)、全例 make <name> green。 弾性波も Einstein 添字記法で .fe 化済み。未移行は Yee/音響(ε_ijk・スカラー対象の添字和で近日)・LBM(family)・MHD(ユーザ定義ヘルパ)のみ。 設計は DSL-DESIGN.md、全19例は 総合ギャラリーへ。

Maxwell — E・B はベクトル場

∂tE = ∇×B,  ∂tB = −∇×E  (c=1)
.fe(28行): field E : vector と宣言すれば、物理は E' = E + dt * curl B / B' = B - dt * curl E' の **添字も成分もない2行**。E' は更新済み配列への参照 (symplectic・袖幅1)。dimension/axes も宣言可能。
DSL ソース全文(maxwell3d.fe・24 行) 
-- Maxwell's equations in vacuum (c = 1), collocated grid
--   dE/dt =  curl B
--   dB/dt = -curl E'     (the B update uses the updated E': symplectic)
-- E and B are vector fields and the equations are whole-vector
-- equations: no components, no indices.

dimension 3
axes x, y, z

param dt = 0.1*dx

extern exp
raw gauss1 = fun(x) exp(-((x-(total_grid_x*dx/2))/(8*dx))**2)

field E : vector
field B : vector

init:
  E = [| 0, gauss1(i*dx), 0 |]
  B = [| 0, 0, gauss1(i*dx) |]

step:
  E' = E + dt * curl B
  B' = B - dt * curl E'

Maxwell — 微分形式(DEC)

∂tB = −dE,  ∂tE = ⋆d⋆B   (E ∈ Ω¹: 辺、B ∈ Ω²: 面)
.fe(27行): field B : 2-form と宣言すれば E' = E + dt * delta B(δ = 余微分 = ⋆d⋆)・B' = B - dt * d E' の2行が物理の全て。ライブラリでは δ を教科書どおり codiff = (−1)^(n(k+1)+1) hodge∘dForm∘hodge で合成(Egison 本体の Yang–Mills サンプルの連続版 δ と同じ構造)。assert-dd-zero E' で d∘d=0 の CAS 検査を通らない限り生成しない。
DSL ソース全文(maxwell_dec.fe・23 行) 
-- Maxwell's equations as discrete differential forms (DEC)
--   Faraday :  dB/dt = -d E'          (d : 1-form -> 2-form)
--   Ampere  :  dE/dt = delta B        (codifferential: delta = *d* on 2-forms)
-- No grid placements appear anywhere: the form degrees determine the
-- Yee lattice, and generation is gated on the CAS reducing d(dE') to 0.

param dt = 0.5*dx

extern exp
raw gauss1 = fun(x) exp(-((x-(total_grid_x*dx/2))/(8*dx))**2)

field E : 1-form
field B : 2-form

init:
  E = [| 0, gauss1(i*dx), 0 |]
  B = [| 0, 0, gauss1((i+0.5)*dx) |]

step:
  E' = E + dt * delta B
  B' = B - dt * d E'

assert-dd-zero E'

弾性波 — 添字記法とテンソル場

ρ∂tvi = ∂jσij,   ∂tσij = λδij∇·v′ + μ(∂iv′j+∂jv′i)
.fe(39行): 物理は数式そのままの Einstein 記法2行 — v'_i = v_i + (dt / rho0) * d_j s_i_j / s'_i_j = s_i_j + dt * (la * delta_ij * d_k v'_k + mu * (d_i v'_j + d_j v'_i))。 繰り返し添字は自動で総和、δ_ij は Kronecker、@ staggered 宣言により d_a は対象成分の配置にアンカーされた半セル差分になる(Virieux 格子が導出される)。
DSL ソース全文(elastic3d.fe・40 行) 
-- Linear elastic waves, velocity-stress formulation (Virieux), written
-- exactly as the mathematics:
--
--   rho dv_i/dt      = d_j sigma_ij
--   d sigma_ij / dt  = la delta_ij (div v') + mu (d_i v'_j + d_j v'_i)
--
-- Repeated index letters are summed (Einstein convention); delta_ij is
-- Kronecker's delta; v' refers to the updated velocity (symplectic).
-- The fields are declared staggered, so every d_a becomes a half-cell
-- difference anchored at the placement of the target component --
-- which puts v_i on edge i and s_ij at edge i (+) edge j: the Virieux
-- lattice, derived rather than declared.  With la=2, mu=1, rho0=1 the
-- P and S speeds are 2 and 1, measured by the check driver.
