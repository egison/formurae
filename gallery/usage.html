Formurae 使い方ガイド

テンソル添字記法、微分形式、解析微分をEgisonで評価し、離散化だけをpost-fecで行って、 FormuraのMPI・temporal-blocking付きC codeへつなぐための手引きです。 実例はFormuraeギャラリーと総合ギャラリーにあります。

目次: pipeline / setup / tutorial / syntax / 演算子の分類 / 微分と精度 / tensor・form / geometry / 生成物と検証

1. 4段pipeline

model.fme    Formurae source
   │ pre-fec: parse / scope / declaration / source map
model.egi    Egison normalization unit
   │ Egison: tensor/index algebra / analytic derivative / CAS
model.feir   canonical Formurae Egison IR
   │ post-fec: placement / exact stencil / auxiliary fields / storage
model.fmr    Formura source
   │ Formura
model.c/.h   MPI + temporal blocking C library

pureな解析式で定義できる演算子はEgison、具体的な差分、格子配置、storageを含む離散意味は opaque requestとpost-fec、loopと通信はFormuraが担当します。 成功時のEgison stdoutはFEIR 1個だけです。warningやtype/evaluation errorはmachine runnerが拒否します。

2. セットアップ

# GHC 9.6系、cabal、stack(Formura用)、隣接する ../egison が必要
make setup
cabal build
make diffusion3d
make all

FormuraeとEgisonはcabalでbuildします。Formura upstreamのbuildだけにstackを使います。 MPIがない環境向けに1-rank stubを同梱しています。

3. 拡散方程式

mode collocated
dimension 3
axes x, y, z

param κ = 1.0
param dt = 0.1*dx*dx

raw gauss = fun(x,y,z) exp(-((x-32*dx)/(5*dx))**2 - ...)

field u : scalar

init:
  u = gauss(i*dx,j*dy,k*dz)

step:
  u' = u + dt * κ * Δ u

make diffusion3dで4つの中間artifactを生成し、FormuraとC checkまで実行します。 Δはmode collocatedのcanonical scalar Laplacianで、離散化精度はmodel profileから選ばれます。

縮約はcontractWithまたは.で明示します。同じ文字の上下添字が現れただけでは surfaceで暗黙総和しません。唯一の例外は記号添字つき∂で、微分が導入した添字をoperandが 繰り返す場合に限りcontractWith (+)を自動適用します(∂_i X~iは発散、 ∂_j σ~i~jはvector)。withSymbols、tensorMap、subrefs、 transpose、!.はEgison tensor algebraへ渡されます。

5. 演算子は「pureに短く定義できるか」で分ける

「基本演算子か合成演算子か」ではなく、演算子の意味そのものを通常のEgison数式として 1行から数行で定義できるかで分けます。pureな演算子はtensor/index algebraと解析微分によって FieldJetを含む式までEgisonが正規化し、その後のstencil選択だけをpost-fecに任せます。

組み込みとライブラリの早見表

「何が組み込みで何がライブラリ関数か」は次の4層で決まります。第1層だけがcompiler所有で、 ユーザは定義も上書きもできません。それ以外の名前はすべて通常のlexical shadowingに従い、 user定義は最下層と同じcompletion・placement経路を通ります。

層surface意味の所有者shadow
構文(primitive) ∂/∂ e a・∂_a・∂^m_a・∂'^m_a・ バッククォートchain(2段以上)・ resample・添字/variance記法・let/localと@policy pre-fecの構文解析+Egisonの∂/∂(未知の微分則はerror)+opaque manifest。 解析微分は∂/∂だけ。添字つき∂は常に離散で、 素の一階(軸・記号とも)はplacement誘導radius-1差分(記号添字は軸ごとに並べ 導入した添字の反復を自動縮約)、次数・プライム付きはexplicit-radius中心stencil (radius = クォート数 + 1) 不可
ambient束縛 dimension・coordinates・volume・epsilon・ metric・inverseMetric(+宣言した計量名) pre-fecがmodelごとに一度束縛 modelスコープで予約
canonical keyword d・hodge・δ・Δ・Δ_H 意味論は可視のライブラリ合成そのもの。名前は正規化前に解決され静的kind検査に使われ、 variable metricのδ・Δだけ内部離散planへlowerする 可
ライブラリ関数 grad・dGrad・divg・curl・ hessian・lap lib/formurae-operators.egiの通常の定義 可(user defと同一経路)

∂族はprimitive層の見えている綴りで、ユーザが定義できない唯一の演算子です (FieldJetを作る側だからです)。contractWith・.・withSymbolsは Egison自身の記法機構で、そのままEgisonコードとして通ります。 以下の小節は、この区分を「定義元」と「必要なambient値」の視点から詳細化したものです。

Egisonと同レベルで簡潔に定義できる演算子

次の三分法は、.fmeの名前がどの定義へ解決されるかと、生成時に coordinates・dimension・metricなどのmodel ambient値が必要かで分ける。 数学的な座標不変性そのものではなく、Formuraeでの定義・束縛に必要な文脈による分類である。

1. Egison標準ライブラリからロードする関数（およびcore構文）

3. Formuraeライブラリで定義し、座標系に依存する関数(canonical keywordを含む)

添字つき∂はこの表から外しました — ライブラリ関数ではなく 構文(早見表の第1層)です。この表のうちd・hodge・δ・ Δ・Δ_Hはcanonical keyword(shadow可、正規化前に名前解決)、 grad以下は純粋に通常のライブラリ関数です。

surface演算子必要なambient値短い定義で表す意味
grad、dGrad、divg、hessian、lapcoordinates微分添字のvarianceを保つ微分・縮約の合成
curlcoordinates、dimension、epsilonLevi-Civita tensorと解析微分の合成
dcoordinatesmode decの外微分
hodgedimension、geometry scale基底補集合とmetric係数によるHodge star
δcoordinates、dimension、geometry scalemode decの余微分。constant geometryではdとhodgeのpureな合成
Δcoordinates、dimension、geometry scalecollocated scalarのcanonical Laplacian。constant geometryでは−δ(d u)=divg (grad u)のpureな合成
Δ_Hcoordinates、dimension、geometry scaleconstant geometry上のHodge--de Rham Laplacian d (δ A) + δ (d A)

metric、inverseMetric、volume、epsilon自体は関数ではなく、 model宣言から生成されるambient tensor/valueである。なお、数学的には座標不変なdも、 現在のFormurae実装ではその成分を∂/∂で計算するためambient coordinatesを参照する。

d、hodge、δ、Δ_Hはmode dec、 canonical Δはmode collocatedのscalarに使います。 curlはambientなepsilonとcoordinatesを使う短い定義へ解決されます。

したがって、user operatorも数学の形を保ったまま書けます。ライブラリ演算子の合成に加えて、 ∂/∂をambientのcoordinatesベクトル(添字なし・添字つき)へ適用すれば 解析テンソル微分そのものが書け、いずれも解析(FieldJet)のままprofileが離散化します。

def symmetricGrad X = sym (dGrad X)
def bilaplacian u = lap (lap u)

def gradLike u = ∂/∂ u coordinates         -- 解析gradient(微分軸はanonymous)

def curlLike X =
  withSymbols [i, j, k]
    (epsilon_i~j~k . ∂/∂ X_k coordinates~j) -- 解析curl(library curlと同じ本体)

metric g
field u : scalar
field q_i
field A_i
field X~i

step:
  q' = gradLike u
  X'~i = withSymbols [j] (g~i~j . A_j)
  A'_i = withSymbols [j] (g_i_j . X~j)

同じ演算子を添字つき∂(withSymbols [i] (∂_i u)など)で書くと、 離散要求(placement誘導radius-1差分)のテンソルを組み立てる演算子になり、profileには 従いません(maxwell3dのcurlは意図的にこの離散形で、既定profileでは解析経由と同じCに落ちる ことを実測済み)。

gradLikeを出たanonymous rank-1軸にはfreshな下添字が補われます。 pre-fecはfunction名やbodyからresult varianceを推論せず、ordinary covectorのequation targetへ 直接代入したときだけ、そのanonymous down軸をtargetの下添字へ対応付けます。 したがってE_i + gradLike uのanonymous軸を既存のiへ暗黙に統合しません。 同じ軸で合成する意図は、withSymbols [i] (E_i + (gradLike u)..._i)のように call siteで下添字を明示します。

Formuraeはuser function本体の式構造や計算履歴を解析してresult contractを付けません。 EgisonがRHSを評価した後、equationまたはindexed localへ格納するときに、宣言targetの shape・logical variance・dfOrderと実際の値だけを照合します。添字の上げ下げは、上の X'~iとA'_iのように計量との縮約をindexed targetへ直接書くのを標準形とします。 Formuraeは関数headの結果添字も、public operatorとしてのflat / sharpも 提供しません。

dimension、coordinates、metric、inverseMetric、 volume、epsilonはmodelから束縛されるambient値です。 metric gを宣言すると、同じ計量を共変なg_i_j / g_#_#と 反変なg~i~jから参照できます。反変viewはindexed equation/localで直接使います。 hidden contextをuser operatorの引数として渡す必要はありません。

格子意味を伴うsurface

数式にstencil、絶対格子位置、保存flux、または一時storageが必要な場合は、 次のsurfaceが意図を構造的に保持します。必要な部分だけがversion付きrequestとしてFEIRへ残ります。

surface格子上の意味
∂_x expr・∂_i expr(記号添字)式全体へのplacement誘導radius-1差分。記号添字は軸ごとに並べ、導入添字の反復を自動縮約
∂^m_x expr・∂'^m_x expr式全体へのexplicit-radius中心stencil(radius = クォート数 + 1)。profileをbypass
`(∂_y (`(∂_x expr)))内側からの軸順と重複を保つ離散微分chain(2段以上。単発のクォートはエラー)
resample(value, bit...)補間先の絶対placement
local q_i @ primal = ...中間grid fieldの型、component placement、lifetime、schedule
geometry付きΔ / δ保存流束またはweighted discrete adjoint、計量係数、補助field

canonical Δはcollocated scalar、δとΔ_HはDEC form専用です。 variable geometryのδとscalar Δは内部の離散planへlowerします。 general variable-metric Δ_Hは現IRでは表せないためcompile-time errorとし、 constant geometryのみd δ + δ dのpureな合成として扱います。

座標非依存の縮約はegison/lib/math/algebra/tensor.egi、 rank-2演算はegison/lib/math/algebra/matrix.egi、 wedgeはegison/lib/math/geometry/differential-form.egiを唯一の定義元とします。 解析微分のcoreはegison/lib/math/analysis/derivative.egi、 model-local calculus・form・geometry定義はlib/formurae-operators.egiにあります。 opaque operatorの権威的一覧はspec/feir-primitives-v1.sexp、 格子programへのloweringはCompile.hsと BackendPlan.hsにあります。

6. 微分と離散化精度

解析微分

∂/∂ (u * u) x   -- 2*u*FieldJet(u,{x:1})、離散化はprofileが決める
∂/∂ (∂/∂ u x) x -- FieldJet(u,{x:2})
∂/∂ u coordinates -- 微分軸を持つテンソル(軸はanonymous、= gradの中身)

解析微分は∂/∂ e a(Egisonと同じ綴り)だけです。Egisonが product/quotient/chain ruleを適用し、導関数はFunctionDataのuser indexから FEIRのFieldJet multi-indexへ変換され、stencilはmodel profileが選びます。 grad・curl・Δなどのライブラリ演算子は この∂/∂の解析合成なので、profileの精度切替はそれら全てに一括で効きます。

添字つき∂ = 離散微分

∂_x (u * u)    -- 式全体に半径1の差分(placement誘導つき、展開しない)
∂^2_x u        -- 3点2階の中心stencil(radius 1)
∂'^2_x u       -- 5点2階の中心stencil(radius 2 = クォート数 + 1)
∂_i X~i        -- 軸ごとのplacement誘導radius-1差分を自動縮約(= 離散div)
∂_j σ~i~j      -- 同上。staggered場では半セルYee差分がplacementから導出される

添字つき∂は添字の種類によらず離散要求です。素の一階は placement誘導のradius-1差分(derivative.grid-whole@1)で、collocatedでは 中心差分、staggeredでは半セルYee差分になります。記号添字はこれを軸ごとに並べた テンソルで、微分が導入した添字をoperandが繰り返す場合に限り contractWith (+)を自動適用します。次数・プライム付きは explicit-radiusの中心stencil(derivative.coordinate-wide@1、radius = クォート数 + 1、 model profileと独立にpost-fecがTaylor moment条件からexact rational係数を解く)。 既定profile(accuracy 2)ではこれらは∂/∂経由と同じstencilに落ちます (elastic3d/maxwell3d/diffusion1d等で生成C byte一致を確認)。

model profile

discretization collocated derivative 2 centered accuracy 4
u' = u + dt * Δ u

精度は演算子本文と独立です。post-fecがformal accuracyを満たす最小radiusを探索し、 Taylor moment条件からexact rational coefficientを解きます。この例はcompact 5点二階差分になります。

順序つき微分chain

`(∂_y (`(∂_x (`(∂_x q))))  -- x, x, yの順に適用

バッククォートは2段以上の微分chain専用です。軸の適用順と重複をそのまま 1個のversioned requestとして保存し(Schwarz正準化を適用しない)、model profileを bypassします。単発のクォート`(∂_x e)は素の∂_x eと 同義になるためエラーです。

配置と中間storage

resample(q, 0, 1)   -- 絶対placementへ線形補間

field u : scalar @ primal
step:
  local q_i @ primal = [| -κ * ∂_x u, -κ * ∂_y u |]_i
  u' = u - dt * divg q

resampleは暗黙補間を避けて絶対placementを明示するときだけ使います。 中間fieldは型付きlocalで宣言します。上のq_i @ primalは成分ごとのface flux、 divg qはcellへ戻る保存差分です。telescopingの保存保証は周期境界、 またはfluxと整合するghost/boundary処理の下で成り立ちます。 物理境界条件は現在もFormura側のYAML・boundary設定が権威です。

7. Tensor、form、placement

Collocatedはcell位置を保ちます。Primal/Dualでは、policyとcomponent basisの parityから各軸の半セルbitを求めます。微分は自然なtarget placementへ移り、異なるplacementを 加算すると明示errorになります。暗黙補間はしません。

mode dec
dimension 3
axes x, y, z

field E : 1-form
field B : 2-form

step:
  E' = E + dt * δ B
  B' = B - dt * d E'

assert-dd-zero E'

mode decではd、hodge、δ、Δ_Hを使います。 constant geometryのΔ_Hはd (δ A) + δ (d A)、variable geometryのδは weighted discrete-adjoint requestとしてpost-fecが補助fieldへlowerします。 general variable-metric Δ_Hは現時点でcompile-time errorです。 assert-dd-zeroはEgison normalizationでd(d E') = 0を確認します。

8. 直交geometryとcanonical Δ

axes θ, φ, z
embedding [ `(2 + cos θ) * cos φ, `(2 + cos θ) * sin φ, sin θ, z ]

u' = u + dt * Δ u

metric scale [...]ではLamé factorを直接指定し、embedding [...]ではEgisonが induced metricを導出して直交性をgateします。metric、inverse、scale、volumeはGeometryNFへ入ります。

geometry付きのcanonical Δは係数field、cell-to-face gradient、保存されるflux、 face-to-cell divergence、volume除算を必要とするため、Egisonは意味を内部requestとして保持します。 post-fecのBackendPlanがstorageとscheduleを決めます。

9. 生成artifactと検証

.fmeが編集対象です。FME exampleではreview用に次を追跡します。

  1. .egi: pre-fec output
  2. .feir: Egison normalization result
  3. .fmr: post-fec output
python3 gallery/tools/embed_src.py  # galleryへ4段sourceを再埋め込み
make all                           # 全exampleのC numerical check

FEIR codec/fingerprint、source diagnostic、effect、operator、placement、Taylor stencil、geometry-aware Δ/δ、 end-to-end pipelineをfocused testで検査します。Egison本体を変更した場合はrepresentative math samples、 mini-test全件、cabal testも実行します。

責務境界、FEIR protocol、検証方法は READMEのcompiler pipelineとrepository構成を参照してください。
