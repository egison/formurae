Formurae 使い方ガイド

テンソル添字記法、微分形式、解析微分をEgisonで評価し、離散化だけをpost-fecで行って、 FormuraのMPI・temporal-blocking付きC codeへつなぐための手引きです。 実例はFormuraeギャラリーと総合ギャラリーにあります。

目次: pipeline / setup / tutorial / syntax / 微分と精度 / tensor・form / geometry / 生成物と検証

1. 4段pipeline

model.fme    Formurae source
   │ pre-fec: parse / scope / declaration / source map
model.egi    Egison normalization unit
   │ Egison: tensor/index algebra / strict analytic derivative / CAS
model.feir   canonical Formurae Egison IR
   │ post-fec: placement / exact stencil / auxiliary fields / storage
model.fmr    Formura source
   │ Formura
model.c/.h   MPI + temporal blocking C library

数学的な演算子はEgison、具体的な差分と格子配置はpost-fec、loopと通信はFormuraが担当します。 成功時のEgison stdoutはFEIR 1個だけです。warningやtype/evaluation errorはmachine runnerが拒否します。

2. セットアップ

# GHC 9.6系、cabal、stack(Formura用)、隣接する ../egison が必要
make setup
cabal build
make diffusion3d
make all

FormuraeとEgisonはcabalでbuildします。Formura upstreamのbuildだけにstackを使います。 MPIがない環境向けに1-rank stubを同梱しています。

3. 拡散方程式

mode collocated
dimension 3
axes x, y, z

param κ = 1.0
param dt = 0.1*dx*dx

def Δ u = divg (grad u)
raw gauss = fun(x,y,z) exp(-((x-32*dx)/(5*dx))**2 - ...)

field u : scalar

init:
  u = gauss(i*dx,j*dy,k*dz)

step:
  u' = u + dt * κ * Δ u

make diffusion3dで4つの中間artifactを生成し、FormuraとC checkまで実行します。 Δは通常のuser-defined Egison functionです。成分展開やstencil callbackを含みません。

5. 微分と離散化精度

解析微分

∂_x (u * u)     -- 2*u*FieldJet(u,{x:1})
∂^2_x u        -- FieldJet(u,{x:2})

通常の∂はEgisonがproduct/quotient/chain ruleを適用する解析微分です。 導関数はFunctionDataのuser indexからFEIRのFieldJet multi-indexへ変換されます。

model profile

discretization collocated derivative 2 centered accuracy 4
def Δ u = divg (grad u)

精度は演算子本文と独立です。post-fecがformal accuracyを満たす最小radiusを探索し、 Taylor moment条件からexact rational coefficientを解きます。この例はcompact 5点二階差分になります。

式全体の離散微分

gridD_x (u * u / 2)

保存形など、積を解析的に展開せず式全体を同じoffsetでsampleしたい場合に使います。 これはversioned opaque requestで、model profileをbypassします。per-occurrence radius overrideの ∂'も同じく明示的なlow-level requestです。

その他の明示離散primitive

orderedD(q, x, y)   -- whole derivativeのaxis順を保持
resample(q, 0, 1)   -- 絶対placementへ線形補間
fluxDiv(F)          -- Primal face fluxの保存divergence
materialize(q)      -- 型・配置を保つstep-local barrier

これらは数学的合成では表せない離散意味を明示するときだけ使います。通常の grad/divg/lapには不要です。lbや可変計量 codiffの補助fieldはpost-fecが自動で計画します。

6. Tensor、form、placement

Collocatedはcell位置を保ちます。Primal/Dualでは、policyとcomponent basisの parityから各軸の半セルbitを求めます。微分は自然なtarget placementへ移り、異なるplacementを 加算すると明示errorになります。暗黙補間はしません。

mode dec
dimension 3
axes x, y, z

field E : 1-form
field B : 2-form

step:
  E' = E + dt * codiff B
  B' = B - dt * d E'

assert-dd-zero E'

d、orthogonal-metric hodge、constant-metric codiffは短いpure Egison definitionです。 variable-metric codiffはweighted discrete-adjoint requestとしてpost-fecが補助fieldへlowerします。 assert-dd-zeroはEgison normalizationでd(d E') = 0を確認します。

7. 直交geometryとlb

axes θ, φ, z
embedding [ `(2 + cos θ) * cos φ, `(2 + cos θ) * sin φ, sin θ, z ]

def Δ u = lb u

metric scale [...]ではLamé factorを直接指定し、embedding [...]ではEgisonが induced metricを導出して直交性をgateします。metric、inverse、scale、volumeはGeometryNFへ入ります。

lbは係数field、cell-to-face gradient、materialized flux、face-to-cell divergence、volume除算を 必要とするため、Egisonでは短いopaque requestです。post-fecのBackendPlanがstorageとscheduleを決めます。

8. 生成artifactと検証

.fmeが編集対象です。FME exampleではreview用に次を追跡します。

  1. .egi: pre-fec output
  2. .feir: Egison normalization result
  3. .fmr: post-fec output
python3 gallery/tools/embed_src.py  # galleryへ4段sourceを再埋め込み
make all                           # 全exampleのC numerical check

FEIR codec/fingerprint、source diagnostic、effect、operator、placement、Taylor stencil、geometry/lb、 end-to-end pipelineをfocused testで検査します。Egison本体を変更した場合はrepresentative math samples、 mini-test全件、cabal testも実行します。

詳しい責務境界とprotocolは pre-fec / post-fec設計書を参照してください。
