Formurae 使い方ガイド

数式のままの表層言語 Formurae(フォーミュレ、拡張子 .fe)で書いた偏微分方程式を、 Egison の計算機代数(CAS)でステンシルに展開し、Formura で MPI+temporal blocking つきの C コードに落とすまでの手引きです。現在の .fe では、∂x のような座標微分、 epsilon~i~j~k . ∂_j X_k のような添字縮約、sin/exp などのスカラー関数を 数式に近い形で書けます。言語名は Formura の複数形= formulae(数式)への掛詞で、 設計者・村主崇行氏の Formura の名を継いでいます。 実例は Formurae ギャラリー(21例)と総合ギャラリー(25例)へ。

1. パイプラインの全体像

model.fe    Formurae ソース: field 宣言+数式(添字記法・微分形式・計量・ユーザ定義演算子)
   │  fec(Haskell; cabal build / cabal run -v0 fec --)
model.egi   埋め込み形(生成中間物; 座標文脈・演算子・プリンタを含む Egison)
   │  Egison の CAS が添字・微分形式・計量を記号的に展開
model.fmr   Formura ソース(全更新式が具体的なステンシルに)
   │  Formura(fork; bin/formura)
model.c/.h  MPI 通信・temporal blocking つき C ライブラリ
   │  cc + 検証ドライバ(*_check.c)
./check     物理検証(保存則・厳密解・参照実装比較; 失敗なら exit≠0)

2. セットアップ

# 前提: GHC 9.6 系+stack(Formura 用)、cabal、../egison に Egison 開発ツリー
make setup     # Formura 2.3.2 を clone + GHC 9.6 パッチ + ビルド → bin/formura
cabal build    # Formurae コンパイラ fec をビルド(依存は base のみ)
make all       # 全25例を生成→実行→検証(数分)

Egison はインストール済みバイナリでなく開発ツリー(EGISON_DIR、既定 ../egison)を 使います — 同梱の数学ライブラリが新しい必要があるためです。MPI は不要(1ランク用スタブ同梱)。

3. チュートリアル: 拡散方程式を動かす

examples/diffusion3d/diffusion3d.fe の全文:

-- The 3D diffusion equation: du/dt = kappa * laplacian(u)

dimension 3
axes x, y, z

param κ = 1.0
param dt = 0.1*dx*dx

extern exp
use exterior-calculus { Δ }
raw gauss = fun(x,y,z) exp(-((x-(total_grid_x*dx/2))/(5*dx))**2 - ...)

field u : scalar

init:
  u = gauss(i*dx,j*dy,k*dz)

step:
  u' = u + dt * κ * Δ u

物理は最後の1行だけです。格子・分割は同じディレクトリの yaml(§5)に書きます。実行:

make diffusion3d
#   fec が .egi を生成 → Egison が .fmr を生成 → formura が C を生成
#   → cc でドライバとリンク → ./check が質量保存を検査して [OK]

式を変えたら(例: u' = u + dt * κ * Δ u - dt * u * ∂x u で移流を足す)、 もう一度 make diffusion3d — 生成された式は examples/diffusion3d/diffusion3d.fmr で確認できます。

4. Formurae 言語リファレンス(.fe)

宣言

構文意味
dimension 1 / dimension 2 / dimension 3次元(必須)。scalar/vector/rank-2 field と k-form は 1D/2D/3D に対応。curl と epsilon~i~j~k は3次元専用として検査する
use exterior-calculus { Δ }座標文脈つき数学演算子を有効化する。Δ は def Δ u = 0 - δ (d u) として追加され、平坦格子では lap、計量宣言下では lb へ下りる
use exterior-calculus { d, δ }外微分・余微分を有効化する。形式場では生成 .egi の座標文脈つき dForm/codiff へ接続する
use vector-calculus { curl, divg }ベクトル解析演算子を有効化する。生成 .egi に座標文脈つき curl/divg を出す。curl は3次元専用として検査する
def Δ u = 0 - δ (d u)ユーザ定義演算子(ファイルスコープ・使用箇所で展開。本文は先に定義された演算子のみ参照可、再帰不可)。use 由来の定義もユーザ定義で上書き可能
def curl X~i = epsilon~i~j~k . ∂_j X_k添字付きユーザ定義演算子の書き方例。上付き/下付き添字を含む引数と、ε 記号・縮約 .・添字微分 ∂_j を使って定義できる(現状は主に1引数演算子)。この curl 定義はまだ use vector-calculus から生成されない
def div X = ∂_i X~i / def grad u_i = ∂_i uスカラーを返す演算子、添字付き成分を返す演算子の例。標準の curl/divg は use vector-calculus で明示した場合だけ、fec が座標文脈つき定義を生成する
axes r, θ, φ軸名(必須)。∂θ の解決や添字文字の走る範囲=座標系を確定させる宣言。生成 Formura の axes ::、格子幅 dr/dtheta、C ドライバの space_interval_r/lower_theta まで宣言名を使う。CAS 内部だけは正規座標 x,y,z に写像する
param dt = 0.5*dxFormura パラメタ(double :: dt = …)。値は Formura 式
extern expextern function :: exp。sin, cos, exp, sqrt, fabs など Formura が扱えるスカラー関数は、式や raw ヘルパで使われたら不足分を自動 extern する。明示 extern も従来通り可
raw 行Formura ソースへそのまま出す行(ヘルパ関数・コメント)
field u : scalarスカラー場
field E : vectorベクトル場。生成 Formura/C の成分名は E_1,E_2,...。成分数は dimension に従い、Formurae ソースには成分名を書かない
field E : k-form微分形式。0 <= k <= dimension を検査し、成分は昇順の軸組で E_1、B_1_2、C_1_2_3 のように出力する。配置は次数と primal/dual から決まる
field v~i @ staggeredスタガード rank-1 tensor。添字の上下は宣言時に固定され、v_i と参照するとエラー
field σ{~i~j} @ staggered対称 rank-2 tensor。{...} は対称、σ^ij は辺i⊕辺j — Virieux 配置が導出される
field A[_i_j] @ staggered反対称 rank-2 tensor。[...] は反対称。独立 storage は上三角 off-diagonal 成分のみ(2D なら1成分、3D なら3成分)で、A_j_i は -A_i_j、A_i_i は 0 に下りる
metric g / metric δ計量テンソルとして使う表層名を宣言する。宣言後は g~i~j/g_i_j や δ_i_j などを書ける。宣言した名前を param/field にも使うとエラー
metric scale [1, 2 + cos theta, 1]直交計量を Lamé スケール因子で直接指定 → lb が使える
embedding [`(1+r) * cos phi, …]座標系の埋め込みから計量を CAS が自動導出(直交性は記号検査で強制)。バッククォートで因子を原子に保つと √ が閉じる
g~i~j / g~i_j / g_i~j / g_i_jmetric g で宣言した計量テンソル。上下添字の形ごとに反変・混合・共変の内部テンソルへ下りる。Euclidean でも単位行列として扱う
assert-dd-zero E'd∘d(E′)=0 を CAS が確認できない限り生成を拒否

init ブロック

構文意味
u = gauss(i*dx,j*dy,k*dz)スカラー場の生の Formura 初期化式
E = [| 0, gauss1(i*dx), 0 |]ベクトル/形式の成分一括指定(要素は Formura 式)
σ~i~j := δ~i~j * exp(x)添字付き CAS 初期化: strict Einstein 検査後に成分展開し、field の配置に合わせて半セル substitute して init に焼き込む
A_i_j = [| [| a12,a13 |], [| a23 |] |]_i_j反対称 rank-2 field の上三角 off-diagonal 初期化。宣言次元の full 反対称行列も受け付ける
u := exp (cos theta - 1)CAS 初期化: 記号式を評価して init に焼き込む(座標は軸名で書く)

step ブロック

構文意味
u' = 式スカラー更新。X' を右辺に書くと「更新後の X」への参照(symplectic な leapfrog が書ける)
E' = E + dt * curl Bベクトル方程式(成分・添字なし)。curl には use vector-calculus { curl } が必要
B' = B - dt * d E'形式方程式(d は次数で解決)。d/δ には use exterior-calculus { d, δ } が必要
v'~i = v~i + (dt/ρ0) * ∂_j σ~i~jテンソル添字方程式(staggered 場)。各項は strict 検査され、自由添字は LHS と上下まで一致、ダミー添字は上1・下1で総和。∂_a は対象成分の配置にアンカーされた半セル差分
local w = ∂x (∂x u)中間格子場(.fmr に1行として出る — 4階微分の2段化などに)
let En_i = …名前つき式(インライン展開される。格子場にはならない)

use / ライブラリ定義

現時点で use exterior-calculus { Δ } が追加する定義は次の1つです。

def Δ u = 0 - δ (d u)

use や計量宣言を持つモデルでは、fec が次のような 座標文脈つき Egison 定義を生成 .egi の先頭に出します。以下は .fe prelude ではなく、dimension/axes から生成されるライブラリ片です。CAS 内部の座標シンボルは x,y,z に正規化し、Formura へ出すときに axes の名前へ戻します。

def feDim : Integer := 3
def feAxes : [String] := ["theta", "phi", "z"]
def feAxisIds : [Integer] := [1, 2, 3]
def feCoords : Vector MathValue := [| x, y, z |]
def feHsteps : Vector MathValue := [| hx, hy, hz |]

def symName (v: String) : String :=
  match v as string with
    | #"hx" -> "dtheta"
    | #"hy" -> "dphi"
    | #"hz" -> "dz"
    | #"x" -> "(i*dtheta)"
    | #"y" -> "(j*dphi)"
    | #"z" -> "(k*dz)"
    | _ -> v

def lap (u: MathValue) : MathValue :=
  sum (map (\a -> dC2 a u) feAxisIds)

def dGrad (X: Vector MathValue) : Matrix MathValue :=
  generateTensor (\[a, b] -> dC a X_b) [feDim, feDim]

def curl (X: Vector MathValue) : Vector MathValue :=
  withSymbols [i, j, k] (ε 3)~i~j~k . (dGrad X)_j_k

def divg (X: Vector MathValue) : MathValue :=
  trace (dGrad X)

def formBasis (k: Integer) : [[Integer]] :=
  if k = 0 then [[]] else if k = 1 then [[1], [2], [3]]
  else if k = 2 then [[1, 2], [1, 3], [2, 3]] else [[1, 2, 3]]

def basisSign (xs: [Integer]) : MathValue := ...  -- permutation sign
def complementBasis (basis: [Integer]) : [Integer] :=
  filter (\a -> not (any (\b -> a = b) basis)) feAxisIds

def hodge (f: (Integer, Integer, [MathValue])) : (Integer, Integer, [MathValue]) :=
  let (c, k, cs) := f
   in let kd := feDim - k
   in (1 - c, kd,
       map (\basis -> let src := complementBasis basis
                       in basisSign (src ++ basis) * formComponent src cs)
           (formBasis kd))

def dForm (f: (Integer, Integer, [MathValue])) : (Integer, Integer, [MathValue]) :=
  let (c, k, cs) := f
   in let kp := k + 1
   in (c, kp,
       map (\basis ->
              foldl (\acc a ->
                       let src := basisRemove a basis
                        in acc + basisSign ([a] ++ src) *
                           dYee a (sigmaC c basis) (formComponent src cs, sigmaC c src))
                    0 basis)
           (formBasis kp))

def codiff (f: (Integer, Integer, [MathValue])) : (Integer, Integer, [MathValue]) :=
  scaleForm ((-1) ^ (feDim * (formDeg f + 1) + 1)) (hodge (dForm (hodge f)))

def δ := codiff

taylorStencil、quote cleanup、形式補助などの座標非依存部分だけが lib/fmrgen.egi に残ります。dTaylor、Δ4、.fmr プリンタは生成 `.egi` 側に出ます。

演算子リファレンス(数式のまま書く)

演算子由来意味
∂x efec 組み込み1階中心差分(軸名で: ∂r, ∂θ, ∂φ …)。座標微分はこの形だけを許可する
∂x (∂x e)fec 組み込み2階コンパクト差分(d の2回適用は半セル前進・後退の対として融合される。d2 という演算子はない)
∂_i X~ifec 組み込み添字微分。axes で宣言した座標方向について展開し、繰り返し添字は上1・下1の Einstein 総和になる
Δ e(= ∇²)use exterior-calculus { Δ }def Δ u = 0 - δ (d u)。平坦なら lap、計量宣言下では lb を使う形に fec が下ろす。use なしの Δ はエラー
lap e生成 .egi の座標文脈定義平坦格子の 2階中心差分和。feAxisIds 上で和を取る
lb efec 組み込み生成計量宣言下の Laplace–Beltrami。hodge 因子の係数場生成・半セル CAS 評価・保存流束まで自動
Δ4 e(= lap4)Egison ライブラリ4次精度 Laplacian。係数は CAS が Taylor 条件を解いて導出
curl X(= ∇ × X)use vector-calculus { curl }回転。表層呼び出しを fec がベクトル式へ展開し、生成 .egi では withSymbols [i,j,k] (ε 3)~i~j~k . (dGrad X)_j_k。use なしの curl はエラー
divg X(= ∇ · X)use vector-calculus { divg }発散。生成 .egi では trace (dGrad X)。use なしの divg はエラー
d ωuse exterior-calculus { d }離散外微分(k-form → (k+1)-form)。形式場では dForm 呼び出しになる。use なしの d はエラー
delta ω(= codiff/δ)use exterior-calculus { δ }余微分 (−1)^{n(k+1)+1} ⋆d⋆。形式場では codiff 呼び出しになる。use なしの δ はエラー

添字: 添字方程式では上付き ~i と下付き _i を常に区別します。添字つき field は field v~i @ staggered、field σ{~i~j} @ staggered、field A[_i_j] @ staggered のように宣言し、rank・上下・対称性・Formura 出力 layout は添字仕様から推論します。初期値も v~i = [| ... |]~i、σ~i~j = [| ... |]~i~j、σ~i~j := δ~i~j * exp(x) のように同じ添字 suffix を明示します。{...} は対称、[...] は反対称です。反対称 field は A_j_i = -A_i_j、A_i_i = 0 に正準化されます。右辺の各項は strict 検査され、自由添字は LHS と上下まで一致し、ダミー添字は上1・下1だけを許します。添字の上げ下げは自動化しないため、必要なら g_i_j * v~j のように metric を明示します。metric g で宣言した計量名は g~i~j/g~i_j/g_i~j/g_i_j を別々の計量テンソルとして展開します。metric δ と宣言すれば、Euclidean 計量を δ_i_j や δ~i~j と書けます。Kronecker delta は metric 名でない場合の mixed identity δ~i_j(1添字1マーク; 融合形 delta_ij は不可)です。例(弾性波): v'~i = v~i + (dt / ρ0) * ∂_j σ~i~j

内部名: metric g の g は生成 Egison で FormuraeInternalMetricContra、FormuraeInternalMetricMixedUpDown、FormuraeInternalMetricMixedDownUp、FormuraeInternalMetricCov に下ります。添字付き field と let A_i のような一時テンソルについても FormuraeInternalTensor...Up/Down... という内部束縛を生成します。FormuraeInternal prefix は予約済みで、ユーザ定義名には使えません。

Unicode: ギリシャ文字(θ, φ, ρ, τ, κ, μ, λ, …)は識別子として使え、 fec が Formura 向けに ASCII 名(theta, phi, …)へ字訳します。 ∂=d、δ=delta(余微分)、−=-。 Δ は use exterior-calculus { Δ } で有効化するモデル依存演算子です(平坦なら lap、計量下なら lb)。 例: トーラス上の熱方程式は axes θ, φ, z + embedding [ `(2 + cos θ) * cos φ, … ] + use exterior-calculus { Δ } + u' = u + dt * Δ u の4行で幾何・使う演算子・物理を書けます (metric_torus.fe がこの形)。

現在の実装範囲: scalar/vector/rank-2/k-form field の 1D/2D/3D 生成、添字付き field の上下・自由添字・ダミー添字検査、indexed CAS initializer、対称/反対称 rank-2 field backend、DEC の一般次元基底生成は実装済みです。 一方、ユーザが定義する一般の添字付きテンソル演算子、多引数テンソル関数は後続作業です。

5. yaml リファレンス(格子と実行構成)

length_per_node: [6.283185307179586, 1.0, 0.0625]   # 物理サイズ(→ dx,dy,dz)
grid_per_node:   [128, 64, 4]                        # 格子点数
mpi_shape:       [1, 1, 1]                           # ランク分割
grid_per_block:  [8, 8, 8]                           # temporal blocking(任意)
temporal_blocking_interval: 4
boundary: [periodic, mirror, fixed 0.0]              # fork 拡張: 軸ごとの境界条件
reduces:  [tot = sum u, umax = max u]                # fork 拡張: 大域リダクション
  • boundary(fork 拡張): periodic / mirror(Neumann)/ fixed v(Dirichlet)。 非周期軸がある場合は temporal blocking なし・mpi_shape [1,…] が必要。
  • reduces(fork 拡張): 毎ステップ全域で集計され n.reduce_<名前> で読める (sum/max/min/absmax)。収束判定・保存則監視・最大値原理の検査に。
  • temporal blocking の制約: grid_per_block ≥ 2×袖幅×interval(違反は fork が検出して拒否)。

6. 検証ドライバの書き方(*_check.c)

#include "モデル名.h"
int main(int argc, char **argv) {
  Formura_Navi n;
  Formura_Init(&argc, &argv, &n);            // init() が走る
  double h = n.space_interval_x;             // axes x,y,z の dx
  double m0 = n.reduce_tot;                  // yaml の reduces
  while (n.time_step < 1000) Formura_Forward(&n);
  for (int i = n.lower_x; i < n.upper_x; i++)  // 内部セルを走査
    ... formura_data.u[i][j][k] ...
  Formura_Finalize();
  return ok ? 0 : 1;                         // 失敗は exit≠0(make が止まる)
}
  • 軸名は .fe の axes 宣言に追随します。たとえば axes r, θ, φ なら n.space_interval_r、n.lower_theta、n.upper_phi のような C 名になります。
  • 周期境界のみのモデルでは、配列内でデータが毎ステップ平行移動します。物理座標は to_pos_x(i, n) で取るか、論理セル ((round(to_pos_x(i,n)/h)) % N + N) % N に写像してから比較します。
  • boundary を使う(アンカー経路)モデルでは移動がなく、生の添字がそのまま物理位置です。
  • 検証は「保存則(流束形式なら機械精度)・厳密解や離散固有値・独立参照実装との比較・ reduces による最大値原理」を組み合わせるのが本リポジトリの流儀です(実例は各 examples/*/*_check.c)。

7. Makefile への登録

新しい例 examples/foo/(foo.fe・foo.yaml・foo_check.c)を追加したら、Makefile の2行だけ:

FE_EXAMPLES  := ... spherical3d foo      # 一覧に追加
CHECK_foo    := foo_check.c              # ドライバ名(必要なら RUNARGS_foo も)

make foo で全段が走ります。生成物(.egi/.fmr は追跡、.c/.h は .gitignore)。

8. ギャラリーの再生成

gallery/gen.sh                    # 各例からデータをダンプ(make all 後に)
python3 gallery/tools/render.py   # PNG/SVG を描画(標準ライブラリのみ; 曲面は 3D 描画)

9. 3層構造と検証の考え方

  • .fe が唯一の編集対象。.egi は生成中間物(ヘッダに GENERATED 印。意味論は 座標文脈・差分コンビネータ・DEC・プリンタを含む。lib/fmrgen.egi は taylorStencil などの小さな core に縮小した。 一部の例(Yee・LBM・MHD・音響)はまだ埋め込み形で直接書かれており、 それらだけ lib/fmrlegacy3d.egi を読む。
  • リファクタや DSL 拡張の際の意味保存は「生成 .fmr のバイト一致」で検証するのが本リポジトリの 流儀 — 記法を変えても同じ機械コードに落ちることが証明になる(例: 弾性波は成分手書き → generateTensor → Einstein 添字 .fe と3世代進化して .fmr は不変)。
  • 数式的な恒等式は生成時に CAS が検査する(DEC の d∘d=0、埋め込みの直交性)。
