Formurae 使い方ガイド

テンソル添字記法、微分形式、解析微分をEgisonで評価し、離散化だけをpost-fecで行って、 FormuraのMPI・temporal-blocking付きC codeへつなぐための手引きです。 実例はFormuraeギャラリーと総合ギャラリーにあります。

目次: pipeline / setup / tutorial / syntax / 演算子の分類 / 微分と精度 / tensor・form / geometry / 生成物と検証

1. 4段pipeline

model.fme    Formurae source
   │ pre-fec: parse / scope / declaration / source map
model.egi    Egison normalization unit
   │ Egison: tensor/index algebra / strict analytic derivative / CAS
model.feir   canonical Formurae Egison IR
   │ post-fec: placement / exact stencil / auxiliary fields / storage
model.fmr    Formura source
   │ Formura
model.c/.h   MPI + temporal blocking C library

pureな解析式で定義できる演算子はEgison、具体的な差分、格子配置、storageを含む離散意味は opaque requestとpost-fec、loopと通信はFormuraが担当します。 成功時のEgison stdoutはFEIR 1個だけです。warningやtype/evaluation errorはmachine runnerが拒否します。

2. セットアップ

# GHC 9.6系、cabal、stack(Formura用)、隣接する ../egison が必要
make setup
cabal build
make diffusion3d
make all

FormuraeとEgisonはcabalでbuildします。Formura upstreamのbuildだけにstackを使います。 MPIがない環境向けに1-rank stubを同梱しています。

3. 拡散方程式

mode collocated
dimension 3
axes x, y, z

param κ = 1.0
param dt = 0.1*dx*dx

def Δ u = divg (grad u)
raw gauss = fun(x,y,z) exp(-((x-32*dx)/(5*dx))**2 - ...)

field u : scalar

init:
  u = gauss(i*dx,j*dy,k*dz)

step:
  u' = u + dt * κ * Δ u

make diffusion3dで4つの中間artifactを生成し、FormuraとC checkまで実行します。 Δは通常のuser-defined Egison functionです。成分展開やstencil callbackを含みません。

5. 演算子は「pureに短く定義できるか」で分ける

「基本演算子か合成演算子か」ではなく、演算子の意味そのものを通常のEgison数式として 1行から数行で定義できるかで分けます。pureな演算子はtensor/index algebraと解析微分によって FieldJetを含む式までEgisonが正規化し、その後のstencil選択だけをpost-fecに任せます。

Egisonと同レベルで簡潔に定義できる演算子

演算子短い定義で表している意味
∂_x、∂^n_x、∂_i、grad、dGrad解析微分と、結果に残る共変微分添字
divg、curl、hessian、lap / user-defined Δ微分、縮約、Levi-Civita tensorの合成
.、contractWith、!. / wedge、trace、sym、antisym、norm2通常のtensor/index algebra
四則演算、sin、exp、sqrtなどscalar algebraと解析的intrinsic function
flat、sharp計量または逆計量との縮約
d、dForm、hodge外微分とHodge star。hodgeの基底補集合と計量係数は共有定義に集約
codiff、delta、δ、formLaplaciandefault Euclidean geometryではdとhodgeのpureな合成

したがって、組み合わせて作るuser operatorも数学の形を保ったまま書けます。

def Δ u = divg (grad u)

def symmetricGrad X = sym (dGrad X)
def bilaplacian u = lap (lap u)

def curlLike X =
  withSymbols [i, j, k]
    (epsilon_i~j~k . ∂_j X_k)

metric g
def raise A =
  withSymbols [i, j]
    (g~i~j . A_j)

def codifferential A =
  let k := dfOrder A
   in (-1) ^ (dimension * (k + 1) + 1)
      * hodge (d (hodge A))

dimension、coordinates、metric、inverseMetric、 volume、epsilonはmodelから束縛されるambient値です。 metric gを宣言すると、同じ計量をg_i_j / g_#_#と g~i~j / g~#~#でも参照できます。 hidden contextをuser operatorの引数として渡す必要はありません。

pureな短いEgison定義だけでは意味が完結しない演算子

次の演算子は、数式だけでなくstencil、絶対格子位置、保存的flux、または一時storageを指定します。 呼び出しは短くても、演算子の意味を上のようなpure Egison functionへ展開することはしません。 Egisonはversion付きのopaque requestを残し、post-fecが格子上のprogramへlowerします。

演算子pureな数式以外に必要な意味
∂'_x、∂'^m_xその出現箇所だけに固定する差分radiusとstencil
gridD_x、gridDerivative_x積・chain ruleを展開せず、式全体を同じoffsetでsampleする離散微分
orderedD、orderedDerivative反復を含む微分軸の順序
resample、interpolate補間先の絶対placement
fluxDiv、conservativeDivmaterialize済みface fluxからの保存的divergence
materialize中間grid fieldのstorage、lifetime、schedule barrier
lb可変係数、cell-to-face gradient、flux storage、face-to-cell divergence、volume除算
geometry付きcodiff、delta、δ、formLaplacianweighted discrete adjoint、dual placement、計量係数のsampleと補助field
def Δ u = lb u   -- 呼び出しは1行だが、lbの意味はopaque requestへ委譲

metric scale [...]またはembedding [...]を宣言したmodelでは、 現状のcodiff系はscale式が定数でもgeometry付きのopaque経路を使います。 またPhase 10aではopaque演算子を1行のuser defから呼べますが、 let/matchを含む複数行body内のopaque callは未対応です。これは上の意味上の分類とは別の実装制約です。

座標非依存の縮約はegison/lib/math/algebra/tensor.egi、 rank-2演算はegison/lib/math/algebra/matrix.egi、 wedgeはegison/lib/math/geometry/differential-form.egiを唯一の定義元とします。 解析微分のcoreはegison/lib/math/analysis/derivative.egi、 model-local calculus・form・geometry定義はlib/formurae-operators.egiにあります。 opaque operatorの権威的一覧はspec/feir-primitives-v1.sexp、 格子programへのloweringはCompile.hsと BackendPlan.hsにあります。

6. 微分と離散化精度

解析微分

∂_x (u * u)     -- 2*u*FieldJet(u,{x:1})
∂^2_x u        -- FieldJet(u,{x:2})

通常の∂はEgisonがproduct/quotient/chain ruleを適用する解析微分です。 導関数はFunctionDataのuser indexからFEIRのFieldJet multi-indexへ変換されます。

model profile

discretization collocated derivative 2 centered accuracy 4
def Δ u = divg (grad u)

精度は演算子本文と独立です。post-fecがformal accuracyを満たす最小radiusを探索し、 Taylor moment条件からexact rational coefficientを解きます。この例はcompact 5点二階差分になります。

式全体の離散微分

gridD_x (u * u / 2)

保存形など、積を解析的に展開せず式全体を同じoffsetでsampleしたい場合に使います。 これはversioned opaque requestで、model profileをbypassします。per-occurrence radius overrideの ∂'も同じく明示的なlow-level requestです。

その他の明示離散primitive

orderedD(q, x, y)   -- whole derivativeのaxis順を保持
resample(q, 0, 1)   -- 絶対placementへ線形補間
fluxDiv(F)          -- Primal face fluxの保存divergence
materialize(q)      -- 型・配置を保つstep-local barrier

これらは数学的合成では表せない離散意味を明示するときだけ使います。通常の grad/divg/lapには不要です。lbや可変計量 codiffの補助fieldはpost-fecが自動で計画します。

7. Tensor、form、placement

Collocatedはcell位置を保ちます。Primal/Dualでは、policyとcomponent basisの parityから各軸の半セルbitを求めます。微分は自然なtarget placementへ移り、異なるplacementを 加算すると明示errorになります。暗黙補間はしません。

mode dec
dimension 3
axes x, y, z

field E : 1-form
field B : 2-form

step:
  E' = E + dt * codiff B
  B' = B - dt * d E'

assert-dd-zero E'

dとorthogonal-metric hodgeは短いpure Egison definitionです。 default Euclidean codiffもその合成です。geometry宣言付きcodiffは weighted discrete-adjoint requestとしてpost-fecが補助fieldへlowerします。 assert-dd-zeroはEgison normalizationでd(d E') = 0を確認します。

8. 直交geometryとlb

axes θ, φ, z
embedding [ `(2 + cos θ) * cos φ, `(2 + cos θ) * sin φ, sin θ, z ]

def Δ u = lb u

metric scale [...]ではLamé factorを直接指定し、embedding [...]ではEgisonが induced metricを導出して直交性をgateします。metric、inverse、scale、volumeはGeometryNFへ入ります。

lbは係数field、cell-to-face gradient、materialized flux、face-to-cell divergence、volume除算を 必要とするため、Egisonは意味を展開せずopaque requestとして保持します。 post-fecのBackendPlanがstorageとscheduleを決めます。

9. 生成artifactと検証

.fmeが編集対象です。FME exampleではreview用に次を追跡します。

  1. .egi: pre-fec output
  2. .feir: Egison normalization result
  3. .fmr: post-fec output
python3 gallery/tools/embed_src.py  # galleryへ4段sourceを再埋め込み
make all                           # 全exampleのC numerical check

FEIR codec/fingerprint、source diagnostic、effect、operator、placement、Taylor stencil、geometry/lb、 end-to-end pipelineをfocused testで検査します。Egison本体を変更した場合はrepresentative math samples、 mini-test全件、cabal testも実行します。

詳しい責務境界とprotocolは pre-fec / post-fec設計書を参照してください。
