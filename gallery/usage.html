Formurae 使い方ガイド

テンソル添字記法、微分形式、解析微分をEgisonで評価し、離散化だけをpost-fecで行って、 FormuraのMPI・temporal-blocking付きC codeへつなぐための手引きです。 実例はFormuraeギャラリーにあります。

目次: pipeline / setup / tutorial / syntax / 演算子の分類 / 微分と精度 / tensor・form / geometry / 生成物と検証

1. 4段pipeline

model.fme    Formurae source
   │ pre-fec: parse / scope / declaration / source map
model.egi    Egison normalization unit
   │ Egison: tensor/index algebra / analytic derivative / CAS
model.feir   canonical Formurae Egison IR
   │ post-fec: placement / exact stencil / auxiliary fields / storage
model.fmr    Formura source
   │ Formura
model.c/.h   MPI + temporal blocking C library

pureな解析式で定義できる演算子はEgison、具体的な差分、格子配置、storageを含む離散意味は opaque requestとpost-fec、loopと通信はFormuraが担当します。 成功時のEgison stdoutはFEIR 1個だけです。warningやtype/evaluation errorはmachine runnerが拒否します。

2. セットアップ

# GHC 9.6系、cabal、stack(Formura用)、隣接する ../egison が必要
make setup
cabal build
make diffusion3d
make all

FormuraeとEgisonはcabalでbuildします。Formura upstreamのbuildだけにstackを使います。 MPIがない環境向けに1-rank stubを同梱しています。

3. 拡散方程式

mode collocated
dimension 3
axes x, y, z

param κ = 1.0
param dt = 0.1*dx*dx

raw gauss = fun(x,y,z) exp(-((x-32*dx)/(5*dx))**2 - ...)

field u : scalar

init:
  u = gauss(i*dx,j*dy,k*dz)

step:
  u' = u + dt * κ * Δ u

make diffusion3dで4つの中間artifactを生成し、FormuraとC checkまで実行します。 Δはmode collocatedのcanonical scalar Laplacianで、離散化精度はmodel profileから選ばれます。

5. 演算子は「pureに短く定義できるか」で分ける

「基本演算子か合成演算子か」ではなく、演算子の意味そのものを通常のEgison数式として 1行から数行で定義できるかで分けます。pureな演算子はtensor/index algebraと解析微分によって FieldJetを含む式までEgisonが正規化し、その後のstencil選択だけをpost-fecに任せます。

組み込みとライブラリの早見表

「何が組み込みで何がライブラリ関数か」は次の4層で決まります。第1層だけがcompiler所有で、 ユーザは定義も上書きもできません。それ以外の名前はすべて通常のlexical shadowingに従い、 user定義は最下層と同じcompletion・placement経路を通ります。

したがって、user operatorも数学の形を保ったまま書けます。ライブラリ演算子の合成に加えて、 ∂/∂をambientのcoordinatesベクトル(添字なし・添字つき)へ適用すれば 解析テンソル微分そのものが書け、いずれも解析(FieldJet)のままprofileが離散化します。

def symmetricGrad X = sym (dGrad X)
def bilaplacian u = lap (lap u)

def gradLike u = ∂/∂ u coordinates         -- 解析gradient(微分軸はanonymous)

def curlLike X =
  withSymbols [i, j, k]
    (epsilon_i~j~k . ∂/∂ X_k coordinates~j) -- 解析curl(library curlと同じ本体)

metric g
field u : scalar
field q_i
field A_i
field X~i

step:
  q' = gradLike u
  X'~i = withSymbols [j] (g~i~j . A_j)
  A'_i = withSymbols [j] (g_i_j . X~j)

同じ演算子を添字つき∂(withSymbols [i] (∂_i u)など)で書くと、 離散要求(placement誘導radius-1差分)のテンソルを組み立てる演算子になり、profileには 従いません(maxwell3dのcurlは意図的にこの離散形で、既定profileでは解析経由と同じCに落ちる ことを実測済み)。

macro — 文列に展開されるテンプレート

defが「式」(Egisonが値に評価)なのに対し、macroは「文列」 (pre-fecが展開)を定義します。本体はlocal束縛の列とin式で、 呼び出しはstep式の中に書けます:

macro Δc u =
  local q~i @ primal = withSymbols [j] (volume * (inverseMetric~i~j . ∂_j u))
  in (divg q) / volume

step:
  u' = u + dt * Δc u

展開はparse直後・全解析の前に行われ、localは fresh 名で呼び出し元stepの 直前へ持ち上げられ(let-insertion)、呼び出し位置にはin式が入ります。 下流の効果・placement・kind検査は展開結果にそのまま働きます。上のΔcは 宣言された直交計量の保存形Laplacianそのもので、生成される.fmrは手書き展開形と 一致します(tests/pre_macro_expansion.sh で検査)。マクロはstep式専用で、 init・def・幾何宣言内での使用はエラーです。

静的検査はscalar/tensorの区別だけを行い、微分形式の次数は値自身 (dfOrder)が正規化時に運びます。local w : tensor @ primal = …と 宣言した遅延localは、rank・variance・次数を書かず、materializeされた値から FEIR宣言を組み立てます。この上で、重みつきflux dFlux (wI = V·∏i∈Igii·AI)と符号込み随伴発散 dFluxDiv((δw)K = −(∏k∈Kgkk/V)·Σi∂iwiK)、 placement誘導grid∂の交代和 dExterior を使うと、 codifferentialが次数によらない1本のマクロになります:

macro δc A =
  local w : tensor @ primal = dFlux A
  in dFluxDiv w

同じ定義が、staggeredトーラス上のスカラーLaplacian (u' = u + dt * (0 - δc (dExterior u))が手書き保存形とlocal名を除いて .fmr一致)にも、flat Yee格子上のMaxwell (E' = E + dt * δc B、2-form操作数!が組み込みδとC実行bit一致)にも 使えます(tests/pre_generic_codiff.sh で検査)。次数分岐はどこにも現れません — ライブラリ関数がdfOrderを読み、遅延localがその次数で実体化します。

gradLikeを出たanonymous rank-1軸にはfreshな下添字が補われます。 pre-fecはfunction名やbodyからresult varianceを推論せず、ordinary covectorのequation targetへ 直接代入したときだけ、そのanonymous down軸をtargetの下添字へ対応付けます。 したがってE_i + gradLike uのanonymous軸を既存のiへ暗黙に統合しません。 同じ軸で合成する意図は、withSymbols [i] (E_i + (gradLike u)..._i)のように call siteで下添字を明示します。

Formuraeはuser function本体の式構造や計算履歴を解析してresult signatureを付けません。 EgisonがRHSを評価した後、equationまたはindexed localへ格納するときに、宣言targetの shape・logical variance・dfOrderと実際の値だけを照合します。添字の上げ下げは、上の X'~iとA'_iのように計量との縮約をindexed targetへ直接書くのを標準形とします。 Formuraeは関数headの結果添字も、public operatorとしてのflat / sharpも 提供しません。

dimension、coordinates、metric、inverseMetric、 volume、epsilonはmodelから束縛されるambient値です。 metric gを宣言すると、同じ計量を共変なg_i_j / g_#_#と 反変なg~i~jから参照できます。反変viewはindexed equation/localで直接使います。 hidden contextをuser operatorの引数として渡す必要はありません。

格子意味を伴うsurface

数式にstencil、絶対格子位置、保存flux、または一時storageが必要な場合は、 次のsurfaceが意図を構造的に保持します。必要な部分だけがopaque requestとしてFEIRへ残ります。

6. 微分と離散化精度

解析微分

∂/∂ (u * u) x   -- 2*u*FieldJet(u,{x:1})、離散化はprofileが決める
∂/∂ (∂/∂ u x) x -- FieldJet(u,{x:2})
∂/∂ u coordinates -- 微分軸を持つテンソル(軸はanonymous、= gradの中身)

解析微分は∂/∂ e a(Egisonと同じ綴り)だけです。Egisonが product/quotient/chain ruleを適用し、導関数はFunctionDataのuser indexから FEIRのFieldJet multi-indexへ変換され、stencilはmodel profileが選びます。 grad・curl・Δなどのライブラリ演算子は この∂/∂の解析合成なので、profileの精度切替はそれら全てに一括で効きます。

添字つき∂ = 離散微分

∂_x (u * u)    -- 式全体に半径1の差分(placement誘導つき、展開しない)
∂^2_x u        -- 3点2階の中心stencil(radius 1)
∂'^2_x u       -- 5点2階の中心stencil(radius 2 = クォート数 + 1)
∂'_x q         -- staggered場では実効radius 3/2 = FDTD(2,4)の4点(±9/8, ∓1/24)
∂''_x q        -- 同、radius 5/2の6点6次
∂_i X~i        -- 軸ごとのplacement誘導radius-1差分を自動縮約(= 離散div)
∂_j σ~i~j      -- 同上。staggered場では半セルYee差分がplacementから導出される

添字つき∂は添字の種類によらず離散要求です。素の一階は placement誘導のradius-1差分(derivative.grid-whole)で、collocatedでは 中心差分、staggeredでは半セルYee差分になります。記号添字はこれを軸ごとに並べた テンソルで、微分が導入した添字をoperandが繰り返す場合に限り contractWith (+)を自動適用します。次数・プライム付きは explicit-radiusの中心stencil(derivative.coordinate-wide、 model profileと独立にpost-fecがTaylor moment条件からexact rational係数を解く)。 幅の規則は「素の形 = 格子の最小stencil、プライム1個 = +1リング」で全格子一様です: collocatedではradius = クォート数 + 1、staggeredの奇数階では結果が双対部分格子に 着地するため半整数の実効radius = クォート数 + 1/2(ターゲット中心 ±1/2 … ±(2k−1)/2 の対称stencil)になります。許される半径は 2r ≡ m (mod 2)(パリティ整合)・点数 2r+1・形式精度 2r − m + 2。 半整数半径そのものは書きません: 表層にもFEIRにも分数は現れず (radius属性は正整数のまま)、post-fecが「staggered×奇数階なら 実効radius = 属性 − 1/2」と解釈するだけです。プライムはパリティ中立な 「+1リング」で、半分ずれるかどうかは格子が供給するため、 「collocatedに3/2」のようなパリティ不整合はそもそも表現できません — placementを宣言でなく(policy, 添字構造)から導出するのと同じ原則です。 既定profile(accuracy 2)ではこれらは∂/∂経由と同じstencilに落ちます (elastic3d/maxwell3d/diffusion1d等で生成C byte一致を確認)。

model profile

discretization collocated derivative 2 centered accuracy 4
u' = u + dt * Δ u

discretization staggered yee accuracy 4   -- staggered場のΔ・grad・divgも4次に

精度は演算子本文と独立です。post-fecがformal accuracyを満たす最小radiusを探索し、 Taylor moment条件からexact rational coefficientを解きます。上の例はcompact 5点二階差分になります。 staggered/yee側のm階則は精度2kの1階半整数stageのm重自己合成で、 Δ = divg∘grad・∂³ = grad∘Δ・∂⁴ = Δ∘Δ が全次数・全精度で恒等的に成立します (accuracy 2ではYee対と[1,−2,1]に退化して従来と一致)。最小幅では合成 = 直接解の 縮退が全次数で成り立ちます。

順序つき微分chain

`(∂_y (`(∂_x (`(∂_x q))))  -- x, x, yの順に適用

バッククォートは2段以上の微分chain専用です。軸の適用順と重複をそのまま 1個のopaque requestとして保存し(Schwarz正準化を適用しない)、model profileを bypassします。単発のクォート`(∂_x e)は素の∂_x eと 同義になるためエラーです。

boundary宣言 — SBP境界閉包(幅は演算子、境界は宣言)

boundary x : sbp                   -- 物理境界: x軸の全微分がSBP閉包つきに
-- boundary y : periodic          -- 既定(宣言なし = 周期 = 従来互換)
-- boundary z : ghost 0.0         -- fork の ghost 充填 idiom の明示化

field u : scalar @ primal
param satx = 2.0*τ*κ/(dx*dx)
step:
  local q @ dual = κ * ∂_x u       -- D⁺: 閉包不要(全行が域内)
  u' = u + dt * (∂_x q             -- D⁻: 両端1行だけSBP閉包行に置換
    + (if x < xlo then 0.0 - satx*u else 0.0)   -- SAT(Dirichlet penalty)
    + (if x > xhi then 0.0 - satx*u else 0.0))

  u' = u + dt * κ * ∂^2_x u        -- 合成2階(= 閉包つきD⁻∘D⁺)を直接書く形

境界の扱いは個々の微分適用の属性ではなく領域(軸)の属性です: boundary x : sbpを宣言すると、その軸に沿う素の∂_x・ ∂^2_xに加え、プライム幅∂'_x(全半径の1階)と profile全accuracyの1–2階もそのまま境界対応になります(k≥2のSBP対は Q=H·D線形化の厳密解として自動構成され、境界次数はk)。内部は通常の staggered差分のまま、境界に最も近い行だけsummation-by-parts閉包行に index guard(if i == 0 then … else …)で置換するため、 どのサンプルも領域外を読みません。閉包はノルム行列Hとの組で 離散部分積分 H_d D⁺ + (H_p D⁻)ᵀ = d_N e_Nᵀ − d₀ e₀ᵀ を厳密に満たすよう選ばれ、 post-fecが有限Nの行列恒等式として毎回検証します(エネルギー法の安定性の根拠)。 エネルギー安定性は「モデル内の全微分が同じ境界扱いを共有する」大域的性質なので、 宣言があるからこそsbp軸に閉包なし経路が混ざったら静的エラーにできます: 幅広の明示2階∂'^2(compactステンシルはD⁻D⁺合成と別物でSBP分解が無い)・ 3階以上・collocated operand・ordered chain横断・half→integerのresampleは compile-time errorです。旧sbpd_x/sbpd2_x綴りは退役し、移行診断が 宣言+素の∂への書き換えを案内します。 境界条件はSAT(penalty項)を表層のif+paramで書きます (係数・しきい値をparamに置くのはdxがCAS側で非束縛のため; H⁻¹端重みは k=1で2/dx・k=2で18/(7dx))。1階はhalf→integerの向きで両端閉包、 integer→halfはk=1のみ閉包不要、2階はinteger配置限定です。実測は examples/sbp_diffusion1d(2.02次収束+エネルギー単調)・ sbp_wave1d(pressure-release、1周期回帰)・ sbp_diffusion2d(軸ごとの加法で角が自動成立)・ sbp_highorder4(内部4次の∂'両方向、実測3.03次収束)。

配置と中間storage

resample(q, 0, 1)   -- 絶対placementへ線形補間

field u : scalar @ primal
step:
  local q_i @ primal = [| -κ * ∂_x u, -κ * ∂_y u |]_i
  u' = u - dt * divg q

resampleは暗黙補間を避けて絶対placementを明示するときだけ使います。 中間fieldは型付きlocalで宣言します。上のq_i @ primalは成分ごとのface flux、 divg qはcellへ戻る保存差分です。telescopingの保存保証は周期境界、 またはfluxと整合するghost/boundary処理の下で成り立ちます。 物理境界条件はFormura側のYAML・boundary設定(ghost充填)が権威で、 有界領域で境界まで含めた保証が要る場合は上のSBP+SATを使います。

7. Tensor、form、placement

Collocatedはcell位置を保ちます。Primal/Dualでは、policyとcomponent basisの parityから各軸の半セルbitを求めます。微分は自然なtarget placementへ移り、異なるplacementを 加算すると明示errorになります。暗黙補間はしません。

成分射影 — q_x と q_i

boundary x : sbp
boundary y : sbp
field q_i @ primal
step:
  local w_i @ primal = [| κ * ∂_x u, κ * ∂_y u |]_i
  u' = u + dt * (∂_x w_x + ∂_y w_y)   -- w_x = wのx成分(x面に住むスカラー)

添字の判定規則は∂と同じです: 宣言済みの軸名なら具体成分、 それ以外は記号添字。q_xは宣言済みの添字つきfield/localのx成分を スカラーとして取り出し(T_x_yはrank-2の具体成分)、placementは従来どおり (policy, 成分basis)から導出されます — 上のw_xは[½, 0]に住み、 sbp軸の∂_xがhalf→integerの閉包つきの向きを自動選択します。対称・反対称の鏡像成分は 束縛テンソル値が符号・零対角込みで持つため、ω_y_xも正しく負値を読みます。 varianceは宣言と一致が必要(q_i宣言に対するq~xはerror、 上下の付け替えは計量との縮約を明示で)。混合添字T_x_jと ランク不一致はcompile-time errorです。射影はRHS専用で、更新はwhole-field targetのままです。

mode dec
dimension 3
axes x, y, z

field E : 1-form
field B : 2-form

step:
  E' = E + dt * δ B
  B' = B - dt * d E'

mode decではd、hodge、δ、Δ_Hを使います。 constant geometryのΔ_Hはd (δ A) + δ (d A)、宣言幾何のδは preludeマクロとして実体化localと公開演算子へ展開されます(§7)。 general variable-metric Δ_Hは現時点でcompile-time errorです。 d(d A) = 0は演算子ライブラリの定理としてcompiler suiteが検査し、 離散的なdiv B = 0の保存はcheck driverが実測します。

8. 直交geometryとcanonical Δ

axes θ, φ, z
embedding [ `(2 + cos θ) * cos φ, `(2 + cos θ) * sin φ, sin θ, z ]

u' = u + dt * Δ u

metric scale [...]ではLamé factorを直接指定し、embedding [...]ではEgisonが induced metricを導出して直交性をgateします。metric、inverse、scale、volumeはGeometryNFへ入ります。

geometry付きのcanonical Δは係数サンプリング、cell-to-face gradient、保存されるflux、 face-to-cell divergence、volume除算を必要とします。宣言幾何ではΔ/δが preludeマクロとして実体化local+公開演算子(dFluxWeights/dFluxScale/dFluxDiv)へ 展開され(§7)、幾何のみの係数localはpost-fecが凍結してinit一回+恒等carryのstate配列にします。
