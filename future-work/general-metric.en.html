1. Summary — deleting the assert is not enough

Formurae currently requires the induced metric of a geometry declared via metric scale [...] / embedding [...] to be symbolically orthogonal (every off-diagonal component reduces to zero in the CAS), and rejects the program otherwise. This check is not a mere guard: the entire downstream pipeline depends on the “metric = diagonal matrix of scale factors” representation — volume = ∏hi, Hodge coefficient = V/∏hi², lb coefficient = V/ha². Removing only the check would make all of these silently produce numbers that ignore the off-diagonal components.

So the real content of “removing the orthogonality requirement” is: build a path that handles general metrics correctly, and change the role of the orthogonality check from a gate into a classifier. The paper (formurae-paper/main.tex, limitations) already names the three requirements: (1) a specified covariant operator, (2) placement rules for cross terms, (3) a new or revised opaque discrete contract. The long-term receptacle is the typed operator graph (weighted-adjoint-d primitive) sketched in design/20260713 §9.3.

4. Why the general-metric Hodge conflicts with staggered DEC

The headline advantage of staggered DEC — a discrete Hodge star that is diagonal, i.e. “multiply one value at the same grid point by a coefficient” — is itself a consequence of metric orthogonality. A staggered grid can still be built over a general metric, but the Hodge turns into an averaging matrix that mixes components living at different placements; it stops being pointwise.

4.1 Why orthogonality makes the Hodge pointwise

In a Yee-style staggered layout each component of a k-form has its own address (figure below: α₁ on x-edge midpoints, α₂ on y-edge midpoints). The orthogonal Hodge is

(⋆α)_J = ± (V / ∏_{i∈I} hᵢ²) · α_I,   I = complement of J

so every target component depends on exactly one source component. Moreover, by Yee duality the primal I-cell and the dual Ic-cell share the same center point. The orthogonal ⋆ therefore reduces to “one value at the same location times a coefficient field” — zero interpolation, exactly invertible. The implementation mirrors this precisely (makeHodgeCoefficient samples a single coefficient at the source placement; the coefficient-field role OrthogonalHodgeCoefficientRole is keyed by a single basis).

4.2 What breaks with a general metric

For 1-forms in 2D, (⋆α)₂ = √g (g¹¹α₁ + g¹²α₂). When the metric is orthogonal, g¹² = 0 and the right-hand side is just α₁ at the same point. The moment g¹² ≠ 0, the target at (i+½, j) demands α₂, which lives at (i, j±½) — a value at a different placement half a cell away, unavailable without a 4-point average. In general the Hodge coefficients are Gram minors and mix all components of the same degree, so the mismatch grows with dimension and degree.

4.3 Three known ways to keep a staggered grid anyway

RouteIdeaCost
Averaged Hodge Define ⋆ as a sparse matrix with neighbor averaging (the lineage of nonorthogonal-grid FDTD, FIT with averaged material matrices, mimetic finite differences) ⋆∘⋆ = ±1 no longer holds exactly; adjointness and energy stability break unless the averaging is designed symmetrically; stencils and halos widen
Lebedev-style component duplication Duplicate every component at every required location so ⋆ becomes pointwise again (standard in staggered schemes for anisotropic elastic waves) Several times the degrees of freedom; spurious modes must be managed
Collocation Put all components at cell centers; a dense ⋆ is trivially pointwise Gives up the staggered advantages (Formurae already has CollocatedMode)

This is not specific to structured grids. In simplicial DEC, too, the diagonal Hodge is the reward for a special structure (circumcentric duals + well-centered meshes); in general one gets the Whitney-form mass matrix, which is non-diagonal. Orthogonality is the structured-grid version of that special structure.

4.4 What this means for Formurae — a contract conflict

As the paper’s analytic/discrete-boundary paragraph states, post-fec’s design principles are: never insert interpolation between incompatible placements implicitly, and make resampling a versioned opaque primitive whenever it is semantically observable. A general ⋆ on staggered form fields applies averaging to fields the user declared, so it would silently cross exactly that boundary. Doing it properly means introducing a new explicit opaque contract that fixes which of the three routes is taken (and, for averaging: which scheme, which direction, how the inverse Hodge is handled) — the generalization of the weighted-adjoint-d graph of design/20260713 §9.3 is the natural receptacle.

The scalar Δ can go first precisely because of the flip side of this argument: the lb source u is collocated and the face fluxes are internal auxiliaries of the contract, so the cross-term averaging can be confined inside lb.general@1 without ever crossing a user-visible placement.

7. Benefit analysis — is keeping the diagonal restriction fine?

The substance of the benefit is: moving specific application classes onto the declarative geometry layer (write the embedding, get metric/volume/coefficients/ verification for free) instead of hand-written discretization. All 27 current examples are served by flat or orthogonal charts, and non-orthogonal problems already have an escape hatch (hand-written explicit fluxes with quoted derivatives). So this is not “impossible → possible”.

7.1 Application classes that need a non-diagonal metric to reach the declarative layer

ApplicationWhy non-diagonal is forcedOrthogonal-chart workaround
Terrain-following (σ) coordinates
weather, ocean, seismic waves		 A shear transformation puts cross terms into the induced metric None in practice (only numerically constructible)
Cubed-sphere
the standard grid of global atmospheric cores		 The gnomonic mapping’s coordinate lines are non-orthogonal A conformal variant exists but has no closed form
Graph surfaces z = f(x,y)
Beltrami flow, membranes, minimal surfaces		 g = I + ∇f∇fᵀ has the fxfy term Isothermal coordinates exist but constructing them is its own problem
Anisotropic diffusion ∇·(D∇u) with cross terms
magnetized-plasma heat conduction, anisotropic porous media		 Off-diagonal D is isomorphic to an off-diagonal metric Only when the grid can be aligned with the principal axes
ALE / moving grids; GR-flavored spatial metrics Deformation and shear generically create cross terms None

The first two matter most. Terrain-following coordinates and the cubed-sphere are the standard charts of Formura’s original home domain (weather and geophysical fluids) and the natural next step after Yin-Yang. Moreover, for both of them the orthogonal substitute charts lack closed forms, so within Formurae’s symbolic-embedding pipeline the workaround charts are themselves unusable: non-orthogonality and the symbolic requirement collide twice over. Without general-metric support there is no route into this domain via declarative geometry.

Anisotropic diffusion comes with a caveat: physically D is a material tensor, not a metric, so whether declaring it as a metric at the surface is the right design is a separate question. But the discrete machinery of lb.general@1 (symmetric coefficient fields Cab, the conservative cross-term scheme) is shared with anisotropic diffusion operators as-is, so the implementation investment pays off for both geometry and materials.

7.2 Relativizations, to avoid overselling the benefit

  • The escape hatch exists. As the minimal-surface design doc does, a non-orthogonal problem can be written today by giving up the metric declaration and hand-writing explicit fluxes. The value of general-metric support is the upgrade from “hand-written” to “declared”. Currently the “the embedding is the only geometric input” claim carries an asterisk: only for charts whose induced metric is diagonal — finding an orthogonal chart is differential-geometry work outsourced to the user.
  • A general Laplacian alone does not make a weather model. The terrain-following and cubed-sphere domains also need advection, Coriolis terms, conservation properties, and so on. Precisely: it opens the chart class; it does not complete the applications.
  • Verification robustness (small but real). The symbolic orthogonality check depends on the CAS’s simplification power; a genuinely orthogonal metric can fail the proof and get rejected. A general path becomes the graceful fallback (hard error → correct but slower general code).

8. Recommendation — the decision criterion

The cost to weigh is not the full generalization but the restricted scope (collocated scalar Δ = lb.general@1 only, staggered general Hodge deferred). That stays within known techniques — cofactor inverses for d ≤ 3, √det g simplification, the standard 9-point conservative scheme — and corresponds to just the core of Phase 4 in §5.

Decision criterion: if terrain-following, cubed-sphere, or graph-surface examples are wanted in the portfolio, the restricted scope is worth it. If not (curved-surface applications stay complete with surfaces of revolution + Yin-Yang), keeping the diagonal restriction is fine — in that case, promote “non-orthogonal problems are written with explicit fluxes” to a documented official policy (e.g. in APPLICATIONS.md), which also stays consistent with the paper’s limitations section as written.
