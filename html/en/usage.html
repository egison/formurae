Formurae Usage Guide

A guide to evaluating tensor index notation, differential forms, and analytic derivatives in Egison, performing only the discretization in formurae-post, and connecting to Formura's MPI, temporal-blocking C code. Worked examples live in the Formurae gallery.

Contents: pipeline / setup / tutorial / syntax / operator classification / derivatives and accuracy / tensor & form / geometry / artifacts and verification

1. The four-stage pipeline

model.fme    Formurae source
   │ formurae-pre: parse / scope / declaration / source map
model.egi    Egison normalization unit
   │ Egison: tensor/index algebra / analytic derivative / CAS
model.feir   canonical Formurae Egison IR
   │ formurae-post: placement / exact stencil / auxiliary fields / storage
model.fmr    Formura source
   │ Formura
model.c/.h   MPI + temporal blocking C library

Operators definable as pure analytic expressions belong to Egison; the discrete semantics — concrete finite differences, grid placement, storage — belong to opaque requests and formurae-post; loops and communication belong to Formura. On success, Egison's stdout is exactly one FEIR. Warnings and type/evaluation errors are rejected by the machine runner.

2. Setup

# Requires GHC 9.6.x, cabal, stack (for Formura), and an adjacent ../egison
make setup
cabal build
make diffusion3d
make all

Formurae and Egison are built with cabal. stack is used only to build Formura upstream. A 1-rank stub is bundled for environments without MPI.

3. The diffusion equation

mode collocated
dimension 3
axes x, y, z

param κ = 1.0
param dt = 0.1*dx*dx

raw gauss = fun(x,y,z) exp(-((x-32*dx)/(5*dx))**2 - ...)

field u : scalar

init:
  u = gauss(i*dx,j*dy,k*dz)

step:
  u' = u + dt * κ * Δ u

make diffusion3d generates the four intermediate artifacts and runs through Formura and the C check. Δ is the canonical scalar Laplacian of mode collocated; the discretization accuracy is chosen from the model profile.

Contraction is written explicitly with contractWith or .. The mere appearance of matching upper and lower indices with the same letter does not trigger implicit summation on the surface. The single exception is ∂ with a symbolic index, which auto-applies contractWith (+) only when the operand repeats an index introduced by the derivative (∂_i X~i is a divergence, ∂_j σ~i~j is a vector). withSymbols, tensorMap, subrefs, transpose, and !. are passed through to Egison tensor algebra.

5. Operators are classified by "does it have a short pure definition?"

The dividing line is not "primitive versus composite operator" but whether the operator's meaning itself can be defined in one to a few lines as an ordinary Egison mathematical expression. Pure operators are normalized by Egison — via tensor/index algebra and analytic differentiation — down to expressions containing FieldJets; only the subsequent stencil selection is left to formurae-post.

Quick reference: built-in versus library

"What is built in and what is a library function" is determined by the following four layers. Only the first layer is compiler-owned; users can neither define nor override it. Every other name follows ordinary lexical shadowing, and user definitions go through the same completion and placement paths as the bottom layer.

LayersurfaceOwner of the semanticsshadow
Syntax (primitive) ∂/∂ e a, ∂_a, ∂^m_a, ∂'^m_a, backquote chains (2+ stages), boundary a : sbp|periodic|ghost v, boundary extrapolation sbpx_a and the SAT macros satDirichlet_a/satNeumann_a, resample, component projection q_x, index/variance notation, let/local and @ policy formurae-pre parsing + Egison's ∂/∂ (unknown derivative rules are errors) + the opaque manifest. Analytic differentiation is ∂/∂ only. Indexed ∂ is always discrete: the bare first order (axis and symbolic indices alike) is the placement-induced radius-1 finite difference (a symbolic index lays it out per axis and auto-contracts repetitions of the introduced index); order/prime forms are explicit-radius centered stencils (collocated: radius = quote count + 1; staggered odd orders: half-integer radius = quote count + 1/2). Boundaries are owned by the axis declaration boundary; on sbp axes the bare ∂ lowers to its SBP-closured form (§6). The ∂ family is the visible spelling of the primitive layer and the only operators users cannot define (they are the side that creates FieldJets). contractWith, ., and withSymbols are Egison's own notation machinery and pass through as Egison code as-is. The subsections below refine this classification from the viewpoints of "where it is defined" and "which ambient values it needs".

Operators definable as concisely as in Egison

The following three-way split is by which definition a .fme name resolves to, and by whether model ambient values such as coordinates, dimension, or the metric are needed at generation time. It classifies by the context required for definition and binding in Formurae, not by mathematical coordinate invariance itself.

metric, inverseMetric, volume, and epsilon themselves are not functions but ambient tensors/values generated from the model declarations. Note that even d, mathematically coordinate-invariant, references the ambient coordinates in the current Formurae implementation because its components are computed with ∂/∂.

User operators can therefore be written while keeping their mathematical shape. Beyond composing library operators, applying ∂/∂ to the ambient coordinates vector (unindexed or indexed) writes the analytic tensor derivative itself; either way the expression stays analytic (FieldJet) and the profile discretizes it.

def symmetricGrad X = sym (dGrad X)
def bilaplacian u = lap (lap u)

def gradLike u = ∂/∂ u coordinates         -- analytic gradient (the derivative axis is anonymous)

def curlLike X =
  withSymbols [i, j, k]
    (epsilon_i~j~k . ∂/∂ X_k coordinates~j) -- analytic curl (same body as the library curl)

metric g
field u : scalar
field q_i
field A_i
field X~i

step:
  q' = gradLike u
  X'~i = withSymbols [j] (g~i~j . A_j)
  A'_i = withSymbols [j] (g_i_j . X~j)

Writing the same operator with indexed ∂ (e.g. withSymbols [i] (∂_i u)) instead yields an operator that assembles a tensor of discrete requests (placement-induced radius-1 finite differences) and does not follow the profile (the curl of maxwell3d deliberately uses this discrete form, and it is measured to compile to the same C as the analytic route under the default profile).

macro — templates that expand into statement sequences

Whereas def defines an "expression" (evaluated to a value by Egison), macro defines a "statement sequence" (expanded by formurae-pre). The body is a sequence of local bindings plus an in expression, and calls can be written inside step expressions:

macro Δc u =
  local q~i @ primal = withSymbols [j] (volume * (inverseMetric~i~j . ∂_j u))
  in (divg q) / volume

step:
  u' = u + dt * Δc u

Expansion happens right after parsing and before all analysis: the locals are hoisted with fresh names to just before the calling step (let-insertion), and the in expression is placed at the call site. Downstream effect, placement, and kind checks act on the expansion result as-is. The Δc above is exactly the conservative-form Laplacian of the declared orthogonal metric, and the generated .fmr matches the hand-written expansion (checked by tests/pre_macro_expansion.sh). Macros are for step expressions only; using one inside init, def, or a geometry declaration is an error.

Static checking distinguishes only scalar versus tensor; the degree of a differential form is carried by the value itself (dfOrder) at normalization time. A deferred local declared as local w : tensor @ primal = … writes no rank, variance, or degree, and assembles its FEIR declaration from the materialized value. On top of this, using the weighted flux dFlux (wI = V·∏i∈Igii·AI), the sign-carrying adjoint divergence dFluxDiv ((δw)K = −(∏k∈Kgkk/V)·Σi∂iwiK), and the alternating sum of placement-induced grid ∂, dExterior, the codifferential becomes a single macro independent of the degree:

macro δc A =
  local w : tensor @ primal = dFlux A
  in dFluxDiv w

The same definition works for the scalar Laplacian on a staggered torus (u' = u + dt * (0 - δc (dExterior u)) matches the hand-written conservative form in .fmr up to local names) and for Maxwell on a flat Yee grid (E' = E + dt * δc B, a 2-form operand! — bit-identical C execution with the built-in δ) (checked by tests/pre_generic_codiff.sh). No degree dispatch appears anywhere — the library functions read dfOrder, and the deferred local materializes at that degree.

The anonymous rank-1 axis coming out of gradLike is completed with a fresh subscript. formurae-pre does not infer a result variance from the function name or body; only when the value is assigned directly to an ordinary covector equation target does it map that anonymous down axis to the target's subscript. It therefore does not implicitly merge the anonymous axis of E_i + gradLike u into the existing i. To combine along the same axis, make the subscript explicit at the call site, as in withSymbols [i] (E_i + (gradLike u)..._i).

Formurae does not attach a result signature by analyzing the expression structure or computation history of a user function body. After Egison evaluates the RHS, only the declared target's shape, logical variance, and dfOrder are checked against the actual value when storing into an equation or an indexed local. For raising and lowering indices, the standard form is to write the contraction with the metric directly into an indexed target, as with X'~i and A'_i above. Formurae provides neither result indices on function heads nor flat / sharp as public operators.

dimension, coordinates, metric, inverseMetric, volume, and epsilon are ambient values bound from the model. Declaring metric g makes the same metric available covariantly as g_i_j / g_#_# and contravariantly as g~i~j. The contravariant view is used directly in indexed equations/locals. There is no need to pass hidden context as arguments to user operators.

Surface forms with grid semantics

When a formula needs a stencil, an absolute grid position, a conserved flux, or temporary storage, the following surface forms preserve the intent structurally. Only the necessary parts remain in the FEIR as opaque requests.

surfaceMeaning on the grid
∂_x expr, ∂_i expr (symbolic index)Placement-induced radius-1 finite difference of the whole expression. A symbolic index lays it out per axis and auto-contracts repetitions of the introduced index
∂^m_x expr, ∂'^m_x exprExplicit-radius centered stencil over the whole expression (radius = quote count + 1). Bypasses the profile
`(∂_y (`(∂_x expr)))Discrete derivative chain preserving inside-out axis order and repetition (2+ stages; a single quote is an error)
resample(value, bit...)Absolute placement of the interpolation target
local q_i @ primal = ...Type, component placement, lifetime, and schedule of an intermediate grid field
Δ / δ with geometryPrelude-macro expansion: materialized weights (init-frozen), flux locals, and whole-operand finite differences

The canonical Δ is for collocated scalars, and Δ_H is for DEC forms only. δ is DEC-form-only under constant geometry, but under declared geometry it becomes a prelude macro (the mode gate of the request era is gone), so it can also be written in collocated mode. Under declared geometry, δ and the scalar Δ lower to conservative flux form via prelude-macro expansion. The general variable-metric Δ_H cannot be expressed in the current IR and is a compile-time error; only under constant geometry is it treated as the pure composition d δ + δ d.

Coordinate-independent contraction has egison/lib/math/algebra/tensor.egi, rank-2 operations have egison/lib/math/algebra/matrix.egi, and wedge has egison/lib/math/geometry/differential-form.egi as its single source of definition. The analytic-derivative core is egison/lib/math/analysis/derivative.egi, and the model-local calculus, form, and geometry definitions are in lib/formurae-operators.egi. The authoritative list of opaque operators is spec/feir-primitives.sexp, and the lowering to grid programs lives in Compile.hs and BackendPlan.hs.

6. Derivatives and discretization accuracy

Analytic derivatives

∂/∂ (u * u) x   -- 2*u*FieldJet(u,{x:1}); the profile decides the discretization
∂/∂ (∂/∂ u x) x -- FieldJet(u,{x:2})
∂/∂ u coordinates -- tensor carrying the derivative axis (the axis is anonymous; = the body of grad)

Analytic differentiation is ∂/∂ e a (the same spelling as Egison) and nothing else. Egison applies the product/quotient/chain rules, derivatives are converted from FunctionData user indices to FEIR FieldJet multi-indices, and the stencils are chosen by the model profile. Library operators such as grad, curl, and Δ are analytic compositions of this ∂/∂, so switching the profile accuracy applies to all of them at once.

Indexed ∂ = discrete derivatives

∂_x (u * u)    -- radius-1 finite difference of the whole expression (placement-induced, not expanded)
∂^2_x u        -- 3-point second-derivative centered stencil (radius 1)
∂'^2_x u       -- 5-point second-derivative centered stencil (radius 2 = quote count + 1)
∂'_x q         -- on staggered fields, effective radius 3/2 = the 4 points of FDTD(2,4) (±9/8, ∓1/24)
∂''_x q        -- likewise, radius 5/2: 6 points, 6th order
∂_i X~i        -- auto-contracts the per-axis placement-induced radius-1 differences (= discrete div)
∂_j σ~i~j      -- same; on staggered fields the half-cell Yee difference is derived from placement

Indexed ∂ is a discrete request regardless of the kind of index. The bare first order is the placement-induced radius-1 finite difference (derivative.grid-whole): a centered difference on collocated grids, a half-cell Yee difference on staggered grids. A symbolic index is the tensor of these laid out per axis, auto-applying contractWith (+) only when the operand repeats an index introduced by the derivative. Order/prime forms are explicit-radius centered stencils (derivative.coordinate-wide; independently of the model profile, formurae-post solves exact rational coefficients from Taylor moment conditions). The width rule — "bare form = the grid's minimal stencil, one prime = +1 ring" — is uniform across all grids: on collocated grids radius = quote count + 1; for odd orders on staggered grids the result lands on the dual subgrid, so the half-integer effective radius = quote count + 1/2 (a symmetric stencil at ±1/2 … ±(2k−1)/2 around the target center). The admissible radii satisfy 2r ≡ m (mod 2) (parity consistency), with 2r+1 points and formal accuracy 2r − m + 2. The half-integer radius itself is never written: no fraction appears in the surface language or the FEIR (the radius attribute stays a positive integer); formurae-post merely interprets "staggered × odd order means effective radius = attribute − 1/2". A prime is a parity-neutral "+1 ring", and whether it shifts by a half is supplied by the grid, so a parity mismatch like "3/2 on collocated" cannot even be expressed — the same principle as deriving placement from (policy, index structure) rather than from a declaration. Under the default profile (accuracy 2) these fall to the same stencils as via ∂/∂ (byte-identical generated C confirmed on elastic3d/maxwell3d/diffusion1d, etc.).

model profile

discretization collocated derivative 2 centered accuracy 4
u' = u + dt * Δ u

discretization staggered yee accuracy 4   -- Δ, grad, and divg on staggered fields become 4th order too

Accuracy is independent of the operator bodies. formurae-post searches for the minimal radius satisfying the formal accuracy and solves exact rational coefficients from Taylor moment conditions. The example above becomes the compact 5-point second-derivative difference. On the staggered/yee side, the order-m rule is the m-fold self-composition of the accuracy-2k first-order half-integer stage, so Δ = divg∘grad, ∂³ = grad∘Δ, and ∂⁴ = Δ∘Δ hold identically at every order and accuracy (at accuracy 2 this degenerates to the Yee pair and [1,−2,1], matching the previous behavior). At minimal width, the collapse "composition = direct solution" holds at every order.

Ordered derivative chains

`(∂_y (`(∂_x (`(∂_x q))))  -- applied in the order x, x, y

The backquote is reserved for derivative chains of two or more stages. It preserves the axis application order and repetitions as-is in a single opaque request (no Schwarz canonicalization is applied) and bypasses the model profile. A single quote `(∂_x e) would be synonymous with the bare ∂_x e and is therefore an error.

boundary declarations — SBP boundary closures (the width belongs to the operator, the boundary to the declaration)

boundary x : sbp                   -- physical boundary: every x derivative gets its SBP closure
-- boundary y : periodic          -- default (no declaration = periodic = legacy-compatible)
-- boundary z : ghost 0.0         -- makes the fork's ghost-fill idiom explicit

field u : scalar @ primal
param sat = τ*κ*sbpHinvX/dx        -- composes the declaration-supplied inverse boundary norm (2/dx)
step:
  local q @ dual = κ * ∂_x u       -- D⁺: no closure needed (every row stays inside the domain)
  u' = u + dt * (∂_x q             -- D⁻: only one row at each end is replaced by the SBP closure row
    + satDirichlet_x(u, 0.0, sat)) -- Dirichlet penalty on both walls (macro)

  u' = u + dt * (∂_x q + satNeumann_x(q, 0.0, 0.0))  -- adiabatic walls: exact replacement of the boundary flux
  u' = u + dt * κ * ∂^2_x u        -- the composed second derivative (= closured D⁻∘D⁺) written directly

Boundary treatment is a property of the domain (the axis), not of individual derivative applications: declaring boundary x : sbp makes not only the bare ∂_x and ∂^2_x along that axis boundary-aware, but also the prime widths ∂'_x (first derivatives at every radius) and first/second derivatives at every profile accuracy (the SBP pairs for k≥2 are constructed automatically as exact solutions of the Q=H·D linearization, with boundary order k). The interior remains the ordinary staggered difference; only the rows closest to the boundary are replaced by summation-by-parts closure rows via an index guard (if i == 0 then … else …), so no sample ever reads outside the domain. Paired with the norm matrix H, the closures are chosen to satisfy the discrete integration by parts H_d D⁺ + (H_p D⁻)ᵀ = d_N e_Nᵀ − d₀ e₀ᵀ exactly, and formurae-post verifies this every time as a finite-N matrix identity (the basis of energy-method stability). Energy stability is a global property — "all derivatives in the model share the same boundary treatment" — so it is precisely the declaration that lets a closure-less path mixed onto an sbp axis be a static error: the wide explicit second derivative ∂'^2 (its compact stencil is distinct from the D⁻D⁺ composition and has no SBP factorization), third order and above, collocated operands, crossing an ordered chain, and half→integer resample are compile-time errors. The old sbpd_x/sbpd2_x spellings are retired, and a migration diagnostic guides the rewrite to declaration + bare ∂. Boundary conditions are SATs (penalty terms), and the sbp declaration supplies named constants: the wall thresholds sbpLoX/sbpHiX and the inverse boundary norm sbpHinvX (= 1/(h_p,0 dx), 2/dx for the minimal pair; models using ∂' or profile accuracy 4 also get sbpHinv4X = 18/(7dx). Axis names are appended capitalized — an underscore would be read as an Egison index). Coefficients live in params composing these (dx is unbound on the CAS side), and the standard forms expand via the per-axis prelude macros satDirichlet_x(u, g, coef) and satNeumann_x(flux, glo, ghi). The boundary values required by Neumann and characteristic SATs are supplied by boundary extrapolation sbpx_x e (extrapolates a dual-placed expression to the wall row via the SBP pair's d₀/d_N; zero in the interior); characteristic SATs (transparent boundaries: left wall sbpHinvX*(p + sbpx_x v'), right wall sbpHinvX*(p - sbpx_x v')) are equation content, so they are written explicitly on the surface. First derivatives close both ends in the half→integer direction, integer→half needs no closure only at k=1, and second derivatives are limited to integer placement. Measured: examples/sbp_diffusion1d (order-2.02 convergence + monotone energy), sbp_wave1d (pressure-release, one-period return), sbp_diffusion2d (corners hold automatically by per-axis additivity), sbp_highorder4 (interior 4th-order ∂' in both directions, measured order-3.03 convergence), sbp_neumann (adiabatic walls, exact heat conservation with 3.4e-15 drift + measured order 2.04), sbp_wave_open (transparent boundaries, no energy growth + mid-flight measured order 1.98).

Placement and intermediate storage

resample(q, 0, 1)   -- linear interpolation to an absolute placement

field u : scalar @ primal
step:
  local q_i @ primal = [| -κ * ∂_x u, -κ * ∂_y u |]_i
  u' = u - dt * divg q

resample is used only to avoid implicit interpolation by making the absolute placement explicit. Intermediate fields are declared with typed local. Above, q_i @ primal is the per-component face flux, and divg q is the conservative difference back to cells. The telescoping conservation guarantee holds under periodic boundaries, or under ghost/boundary handling consistent with the flux. Physical boundary conditions are authoritatively Formura's YAML boundary settings (ghost fill); when a bounded domain needs guarantees up to and including the boundary, use the SBP+SAT above.

7. Tensor, form, placement

Collocated keeps cell positions. Under Primal/Dual, the half-cell bit of each axis is derived from the policy and the parity of the component basis. Derivatives move to their natural target placement, and adding different placements is an explicit error. There is no implicit interpolation.

Component projection — q_x versus q_i

boundary x : sbp
boundary y : sbp
field q_i @ primal
step:
  local w_i @ primal = [| κ * ∂_x u, κ * ∂_y u |]_i
  u' = u + dt * (∂_x w_x + ∂_y w_y)   -- w_x = the x component of w (a scalar living on the x faces)

The index classification rule is the same as for ∂: a declared axis name selects a concrete component, anything else is a symbolic index. q_x extracts the x component of a declared indexed field/local as a scalar (T_x_y is a concrete rank-2 component), and placement is derived as always from (policy, component basis) — the w_x above lives at [½, 0], and ∂_x on an sbp axis automatically selects the closured half→integer direction. Mirror components of symmetric/antisymmetric tensors are carried by the bound tensor value with signs and the zero diagonal included, so ω_y_x correctly reads the negative value. Variance must match the declaration (q~x against a q_i declaration is an error; raising and lowering is written explicitly as contraction with the metric). Mixed indices T_x_j and rank mismatches are compile-time errors. Projection is RHS-only; updates keep whole-field targets.

mode dec
dimension 3
axes x, y, z

field E : 1-form
field B : 2-form

step:
  E' = E + dt * δ B
  B' = B - dt * d E'

mode dec uses d, hodge, δ, and Δ_H. Under constant geometry Δ_H is d (δ A) + δ (d A); under declared geometry δ expands as a prelude macro into materialized locals and public operators (§7). The general variable-metric Δ_H is currently a compile-time error. d(d A) = 0 is checked by the compiler suite as a theorem of the operator library, and conservation of the discrete div B = 0 is measured by the check driver.

8. Orthogonal geometry and the canonical Δ

axes θ, φ, z
embedding [ `(2 + cos θ) * cos φ, `(2 + cos θ) * sin φ, sin θ, z ]

u' = u + dt * Δ u

metric scale [...] specifies the Lamé factors directly; with embedding [...], Egison derives the induced metric and gates on orthogonality. The metric, its inverse, the scales, and the volume enter the GeometryNF.

The canonical Δ with geometry requires coefficient sampling, a cell-to-face gradient, a conserved flux, a face-to-cell divergence, and division by the volume. Under declared geometry, Δ/δ expand as prelude macros into materialized locals plus the public operators (dFluxWeights/dFluxScale/dFluxDiv) (§7), and geometry-only coefficient locals are frozen by formurae-post into state arrays computed once at init and carried by identity.
