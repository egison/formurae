# formura-egison 応用カタログ

Egison のテンソル添字記法で連続系の PDE を 2〜3 行書き、Formura で分散並列+
temporal blocking つきの C に落とす——このパイプラインで狙える応用を、
**実装可能性と検証方法つき**で整理する。生物系(反応拡散の生態解釈)は
`examples/pearson3d` で実証済みなので、ここでは**物理・工学系を中心**に挙げる。

## 適用可能性の判定基準

Formura 2.x + 現行 fmrgen に乗るのは次を満たす問題:

1. **陽的時間発展**(explicit): u^{n+1} が u^n の局所ステンシルで書ける
2. **局所相互作用**: 近接格子点のみ参照(長距離力・大域結合なし)
3. **周期境界で意味がある**(または物理境界をマスク係数場で表現できる)
4. **固定 dt で安定**(最悪波速を事前に見積もれる)

プリンタの現行対応は「多項式 × 格子参照 × 記号」。したがって
**多項式的な非線形項(u·v², |ψ|²ψ など)は今すぐ書ける**が、
超越関数(exp, atan2)や if を含む式は「プリンタ小拡張」枠になる。

凡例 — ◎: 現行機能で今すぐ / ○: プリンタ小拡張(extern 関数・if の出力対応)で可 /
△: Formura 本体拡張(境界条件 or 大域リダクション、[UPSTREAM.md](UPSTREAM.md))後に実用化

## 各テーマの要点

### 1. 理想 MHD — 「次の一手」最有力

Formura 同梱の手書き 283 行 HLLD(`vendor/formura/sample/mhd_3d_hlld`)の簡易版
(Lax–Friedrichs flux)をテンソル記法で:

```egison
def flux (a : Integer) (Us : ...) := ...   -- 保存形 F_a(U)
def lf a q := (shift a 1 (flux a q) + flux a q) / 2
              - lamMax / 2 * (shift a 1 q - q)      -- λmax は double パラメタ
def dBdt := curlYee ... (crossE v B ...)            -- ∂B/∂t = ∇×(v×B)
```

誘導方程式を **constrained transport(B を面配置=Yee 配置)**で書けば
∇·B ≡ 0 が機械精度で保存される(maxwell3d_yee で実証済みの機構をそのまま流用)。
検証は Orszag–Tang 渦 — FHPC'16 論文 Fig.3 と同じ絵が出るはず。
論文の「notation が boilerplate を消す」主張の決定打になる。

### 2. 弾性波(Virieux スタガード格子)— Yee の直系

速度 v_i と応力 σ_ij(対称 6 成分)を半セルずらして配置する標準法。
配置表は `sE`/`sB` と同様に σ の成分ごとに宣言するだけ:

```egison
def sV   := [[1/2,0,0], [0,1/2,0], [0,0,1/2]]         -- v_i
def sSd  := [0,0,0]                                    -- σ_xx,σ_yy,σ_zz(格子点)
def sSxy := [1/2,1/2,0]                                -- σ_xy(辺)ほか
def dvdt i := (1/rho) * sum (map (\j -> dYee j (sV_i) (sigma i j)) [1,2,3])
def dsdt i j := lam * delta i j * divv + mu * (dYee j _ (v i) + dYee i _ (v j))
```

検証: 弾性エネルギー ∫(ρv²/2 + σ:ε/2) の保存、P 波速 √((λ+2μ)/ρ)・S 波速 √(μ/ρ) の実測。
地表自由境界・吸収底面をつけると実務的地震波計算になる(→ UPSTREAM.md の境界条件)。

### 5. Burgers — 解析解つき非線形のベンチマーク

Cole–Hopf 変換で 1D 厳密解が書けるため、**非線形項の生成が正しいことを
解析解との L∞ 誤差で機械検証できる**唯一級の題材。回帰テストに最適。

### 7. 格子ボルツマン(D3Q19)— コード生成の強みが最大化する例

19 本の分布関数 f_k の式は人手では書きたくない。Egison 側で速度集合
e_k と重み w_k を**データ**として持ち、19 本の streaming+collision 式を
`map` で自動生成する。equilibrium の u·e_k, u² は全部多項式なので現行プリンタで通る。
「式の本数が多い系ほど得をする」ことの定量デモ(生成行数/記述行数の比)になる。

### 8. Cahn–Hilliard — 4 階微分と 2 段参照

μ を .fmr の中間変数として出せば各式は半径 1 に保てる(Yee で実証済みの
primed-intermediate 参照と同じ機構)。Σc の厳密保存(丸め誤差のみ)が
強い検証になる。スピノーダル模様は見栄えもよい。

### 9. TDGL 超伝導渦 — 「今すぐ動いて派手」枠

複素場を実 2 場 (a,b) にすると全項が多項式。ランダム初期値から渦(位相欠陥)が
自発形成され、巻き数が量子化される。実装コストは pearson3d 並みに低い。

### 11. 計量つき拡散 — Egison ならではの看板候補

ユーザは計量 g_ij を与えるだけ。√g, g^{ij}, Laplace–Beltrami の展開は
**Egison の CAS が記号的に実行**し、係数が焼き込まれた .fmr が出る。
トーラス(周期境界そのもの!)上の拡散が最初の題材として綺麗。
Devito/GT4Py にはない芸当で、論文の差別化ポイントを増やす。

**実装済**(`examples/metric_torus/`): トーラス R=2, r=1 の誘導計量
√g = 2+cos θ から `sqg := 2 + cos x` の1行で係数 A=√g·g^{θθ}, B=√g·g^{φφ}
を記号導出し、`fmrInit` が係数場 ca/cb/sg と初期値 exp(cos θ+cos φ−2) の
init 本体(`ca[i,j,k] = cos((i*dx)+dx/(2))+2` 等)まで生成。流束形式
(f1/f2 を半セル位置に置く Yee 機構の流用)で計量重みつき熱量が厳密保存。
独立 C 参照実装と 3.3e-16 一致、最大値原理は `reduces` の umax/umin で監視。

### 16. 高次スキーム自動導出 — 応用ではなく「武器の増設」

Taylor 展開 u(x+kh) = Σ u^{(m)} (kh)^m/m! を Egison で記号展開し、
係数の連立方程式を解いて 4 次精度中心差分を**導出ごと**生成する
(`dC2o4` = (−u₂+16u₁−30u₀+16u₋₁−u₋₂)/12h²)。導出が CAS 内にあるので、
任意次数・任意ステンシル幅に一般化できる。全応用の精度が一段上がる。

## 推奨実装順

1. **MHD(Orszag–Tang)** — 論文インパクト最大(283 行 → 数十行、Fig.3 再現)
2. **弾性波(Virieux)** — yeeRef の一般性の実証、Devito の土俵での比較材料
3. **Burgers** — 解析解による機械検証を CI に入れる
4. **Cahn–Hilliard / TDGL** — 低コストで題材の幅(材料・超伝導)を見せる
5. **LBM D3Q19** — 「式の自動生成」の定量デモ
6. 境界条件・大域リダクションの本体入り後: Euler の CFL 適応 dt、地震波の自由地表、PML

いずれも本リポジトリの流儀(生成 .egi + yaml + 検証ドライバ + Makefile ターゲット、
不変量チェックで exit code を返す)に従えば、`make <name>` で再現可能な形になる。
